## 1. 引言

### 1.1 目的
本文件定義戰爭軍備模組（MilitarySystem）的軟體需求，涵蓋兵力構成、戰鬥類型、攻城結構、軍費制度與武將能力影響，作為實作與驗收依據。

### 1.2 範圍
MilitarySystem 負責：
- 兵力類型管理（城兵 / 農兵）
- 三種戰鬥類型的觸發與流程（野戰 / 圍城戰 / 攻城戰）
- 攻城戰的多丸防禦結構與城門機制
- 軍糧與士氣管理
- 軍費分擔制度
- 武將四面向能力對軍事的影響

本 SRS **不涵蓋**武將評價的計算與演化（見 SRS_0011），亦不涵蓋戰後領地處置（見 SRS_0009）。

> **⚠️ 地形系統**（野戰地形效果）為 Phase 4+ 功能，MVP 階段略過，保留 API 鉤點但不實作地形計算。

### 1.3 術語定義

| 術語 | 說明 |
|------|------|
| 城兵 | 城池常備軍，可立即出動 |
| 農兵 | 由郡徵召的民兵，需 1 個月集結 |
| 士氣 | 部隊戰鬥意志，影響戰力與持久力 |
| 軍糧 | 維持軍隊作戰的補給資源 |
| 丸 | 城池的防禦圈層（本丸/一之丸/二之丸等） |
| 本陣 | 指揮官所在的後方陣地 |

## 2. 系統概述

### 2.1 模組定位
MilitarySystem 處理所有與戰鬥相關的邏輯，由 CoreSystem 在軍事階段呼叫。

### 2.2 外部相依模組

| 相依模組 | 相依方式 | 用途 |
|---------|---------|------|
| TerritorySystem | 呼叫 | 取得城等級/城兵上限、農兵可動員量 |
| RetainerSystem | 呼叫 | 取得出兵家臣團與武將資料 |
| RetainerEvalSystem | 呼叫 | 取得武將四面向內部數值 |
| IntrigueSystem | 呼叫 | 軍費全由家臣負擔時降低忠誠度 |
| PostWarSystem | 呼叫 | 戰鬥結束後觸發戰後處置 |
| CouncilSystem | 被呼叫 | 出兵超過總兵力 30% 時自動觸發評議 |

## 3. 功能需求（SA 視角）

### 3.1 兵力構成

**FR-MIL-001**：兵力分兩種來源，特性如下：

| 類型 | 集結時間 | 軍糧負擔 | 士氣特性 | 農業影響 |
|------|---------|---------|---------|---------|
| 城兵 | 立即 | 全由本家負擔 | 穩定（依城等級） | 無 |
| 農兵 | 1 個月 | 農民自負前 3 月 | 較低，依郡軍設施 | 農業收成下滑 |

**FR-MIL-002**：城兵上限 = 城等級 × 500（詳見 TerritorySystem FR-TER-014）。

**FR-MIL-003**：農兵可動員量 = 郡人口 × 0.1（MVP 以郡的軍事設施等級代替人口估算）。

### 3.2 武將能力對戰鬥的影響

**FR-MIL-004**：武將四面向內部數值影響戰鬥計算：

| 面向 | 影響範疇 |
|------|---------|
| 軍事 | 部隊攻擊力、防禦力、士氣消耗效率 |
| 智謀 | 計謀成功率、外交/特務加成 |
| 政治 | 軍糧消耗控制效率（降低耗糧速度） |
| 魅力 | 士氣恢復速度、策反成功率、部隊鬥志加成 |

> 武將評價對外顯示為文字詞彙（見 SRS_0011），戰鬥計算使用內部數值（0–100）。

### 3.3 出兵前置條件

**FR-MIL-005**：出兵超過本家總兵力 30% 時，自動觸發「軍事評議」（CouncilSystem.auto_trigger）。

**FR-MIL-006**：農兵出兵需 1 個月集結期，在集結完成前不能投入戰鬥。

**FR-MIL-007**：出兵時需指定主將（家臣團首領），由其軍事數值決定該軍團的戰鬥表現。

### 3.4 軍糧與士氣管理

**FR-MIL-008**：軍糧每回合消耗，消耗量受以下因素影響：
- 出兵兵力大小
- 行軍距離（MVP 暫以固定消耗率估算）
- 主將政治數值（高政治 → 較低消耗）

**FR-MIL-009**：軍糧耗盡時部隊自動撤退，並觸發士氣崩潰懲罰。

**FR-MIL-010**：士氣耗損來源：
- 每次戰損
- 長期包圍（圍城戰守方）
- 主將陣亡
- 軍糧不足

**FR-MIL-011**：士氣恢復：主將魅力數值越高，每回合士氣恢復量越大。

### 3.5 戰鬥類型

**FR-MIL-012**：三種戰鬥類型觸發條件與流程：

| 類型 | 觸發條件 | 介面風格 | 勝利條件 |
|------|---------|---------|---------|
| 野戰 | 攻擊「郡」 | 戰棋式（類火焰之紋章，六角格）| 敵軍全滅或撤退 |
| 圍城戰 | 攻擊「城」且選擇包圍 | 無戰鬥地圖，回合制消耗 | 守方軍糧耗盡 or 策反降伏 |
| 攻城戰 | 攻擊「城」且選擇強攻 | 多丸指揮介面 | 攻破本丸 |

> **⚠️ 野戰地形效果**（林地、河川等地形加成）：Phase 4 後實作，MVP 不計算地形係數。

### 3.6 攻城戰詳細架構

**FR-MIL-013**：城池防禦採多丸結構（由外至內）：

| 丸 | 城門數 | 守軍特性 | 備註 |
|----|-------|---------|------|
| 城門區（外郭） | 1–2 | 輕兵、陷阱兵 | 初始接觸點 |
| 三之丸 | 1–2 | 輕步兵 | 易攻破，可誘敵 |
| 二之丸 | 2–3 | 中堅兵 + 副將 | 中線防守核心 |
| 一之丸 | 1–2 | 精銳兵 | 城防主力 |
| 本丸 | 1 | 主將與護衛 | 失守即城陷落 |

**FR-MIL-014**：城門規則：
- 城門未破前無法進入該丸
- 守方可下達「關門」、「維修」指令延遲進攻
- 攻方可使用「集中破門」或「火攻」（Phase 4+）

**FR-MIL-015**：攻城戰指揮：
- 玩家操作本陣，每回合可下達 1–3 策令
- 策令選項：進攻區域 / 部隊轉移 / 使用計策 / 士氣鼓舞
- 攻方策略：正面強攻 / 分兵襲門 / 等待內應

**FR-MIL-016**：勝敗條件：

| 角色 | 勝利條件 | 敗北條件 |
|------|---------|---------|
| 攻方 | 攻破本丸 | 軍糧耗盡 / 士氣崩潰 / 主將被擊破 |
| 守方 | 撐過一定回合 / 迫使敵軍撤退 | 本丸失守 / 主將陣亡 / 叛變事件 |

### 3.7 圍城戰

**FR-MIL-017**：圍城戰為自動計算，每回合扣除守方軍糧（扣除速度受主將政治影響）。

**FR-MIL-018**：圍城期間攻方可嘗試派遣忍者籠絡守城家臣（呼叫 IntelligenceSystem）。

### 3.8 軍費分擔制度

**FR-MIL-019**：出兵時大名可設定軍費分擔比例（本家 vs. 出兵家臣團）。

**FR-MIL-020**：分擔比例影響：

| 狀況 | 影響 |
|------|------|
| 全由家臣負擔 | 家臣忠誠度下降（依出兵規模與持續時間） |
| 全由本家負擔 | 本家財政壓力增加（金/糧庫存減少） |

**FR-MIL-021**：忠誠度影響透過 `IntrigueSystem.apply_loyalty_delta()` 執行。

## 5. 資料模型

```
MilitaryForce (Resource)        # 出兵部隊
├── id: String
├── faction_id: String
├── leader_retainer_id: String  # 主將 ID
├── force_type: ForceType       # 城兵 / 農兵
├── size: int                   # 兵力數量
├── morale: int                 # 士氣值 0–100
├── supplies: int               # 攜帶軍糧
├── is_assembled: bool          # 農兵是否完成集結
├── assembly_turns_remaining: int
└── location_id: String         # 所在領地 ID

BattleState (Resource)          # 戰鬥狀態（臨時，戰鬥結束後清除）
├── battle_type: BattleType
├── attacker_force_id: String
├── defender_force_id: String
├── current_turn: int
├── castle_zones: Array[CastleZone]  # 攻城戰各丸狀態
└── outcome: BattleOutcome      # 結果

CastleZone (Resource)           # 攻城戰各丸
├── zone_name: String           # 本丸/一之丸 等
├── gates_total: int
├── gates_broken: int
├── garrison: int               # 守軍兵力
└── is_fallen: bool
```

## 6. GDScript 介面規格（PG 視角）

### 6.1 列舉型別

```gdscript
enum ForceType {
    CASTLE,   # 城兵
    PEASANT   # 農兵
}

enum BattleType {
    FIELD,    # 野戰
    SIEGE,    # 圍城戰
    ASSAULT   # 攻城戰
}

enum BattleOutcome {
    ATTACKER_WINS,
    DEFENDER_WINS,
    STALEMATE,
    RETREAT
}

enum BattleCommand {
    ADVANCE,       # 進攻區域
    TRANSFER,      # 部隊轉移
    USE_TACTIC,    # 使用計策
    BOOST_MORALE,  # 士氣鼓舞
    BREAK_GATE,    # 集中破門
    RETREAT_CMD    # 撤退
}
```

### 6.2 系統節點

```gdscript
class_name MilitarySystem extends Node

# --- 兵力查詢 ---
func get_castle_force_cap(castle_id: String) -> int
func get_peasant_available(district_id: String) -> int
func get_current_garrison(castle_id: String) -> int

# --- 出兵 ---
func recruit_peasants(district_id: String, amount: int) -> Dictionary
    # 回傳: {"success": bool, "force_id": String, "assembly_turns": int}
func deploy_force(retainer_group_id: String, target_id: String, battle_type: BattleType) -> Dictionary
    # 回傳: {"success": bool, "council_required": bool, "message": String}

# --- 軍糧與士氣 ---
func consume_supplies_per_turn(force_id: String) -> int  # 回傳本回合耗糧量
func recover_morale(force_id: String) -> int             # 回傳回復量
func check_supply_exhausted(force_id: String) -> bool

# --- 戰鬥流程 ---
func start_battle(attacker_id: String, defender_territory_id: String, type: BattleType) -> BattleState
func process_battle_turn(battle_state: BattleState, command: BattleCommand) -> Dictionary
    # 回傳: {"events": Array[String], "state": BattleState, "is_over": bool, "outcome": BattleOutcome}
func resolve_siege_turn(battle_state: BattleState) -> Dictionary  # 圍城戰自動計算
func end_battle(battle_state: BattleState) -> Dictionary
    # 回傳: {"outcome": BattleOutcome, "casualties": Dictionary, "trigger_post_war": bool}

# --- 攻城戰 ---
func attack_gate(battle_state: BattleState, zone_index: int) -> Dictionary
func attempt_fire_attack(battle_state: BattleState) -> Dictionary  # Phase 4+，MVP 留鉤點

# --- 武將能力加成計算 ---
func calculate_attack_power(retainer_id: String, base: int) -> int   # 依軍事數值
func calculate_supply_efficiency(retainer_id: String) -> float       # 依政治數值
func calculate_morale_recovery(retainer_id: String) -> int           # 依魅力數值

# --- 軍費 ---
func set_supply_ratio(force_id: String, faction_ratio: float) -> void
    # faction_ratio: 0.0–1.0（本家負擔比例）
func calculate_loyalty_penalty(force_id: String) -> int
    # 計算家臣因軍費負擔而降低的忠誠 delta

# --- 地形（Phase 4+ 保留鉤點）---
func get_terrain_modifier(territory_id: String) -> float  # MVP 固定回傳 1.0
```

### 6.3 訊號


## 7. 驗收準則（DoD 對應）

| 驗收項目 | 判定方式 |
|---------|---------|
| 城兵/農兵兵力上限正確 | 單元測試 |
| 農兵需 1 個月集結才可參戰 | 手動測試 |
| 出兵 > 30% 自動觸發評議 | 手動測試 |
| 攻城戰城門/丸結構正確 | 手動測試 |
| 軍糧耗盡自動撤退 | 手動測試 |
| 武將政治降低耗糧速度 | 單元測試（比較不同政治值的耗糧差異） |
| 地形函式回傳 1.0（MVP） | 單元測試 |
