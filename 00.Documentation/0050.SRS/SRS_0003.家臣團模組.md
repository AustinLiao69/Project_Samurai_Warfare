## 1. 引言

### 1.1 目的
本文件定義家臣團模組（RetainerSystem）的完整軟體需求，涵蓋組織架構、兵力配置、任命邏輯與指揮規則，作為實作與驗收的依據。

### 1.2 範圍
家臣團模組負責：
- 家臣的資料管理（階級、兵力、系譜分類）
- 三層級組織架構（本家家臣團 / 一級家臣團 / 二級家臣團）
- 任命、晉升、解除、回收邏輯
- 兵力計算與指揮封閉性
- 出兵單位管理

本 SRS **不涵蓋**忠誠度計算邏輯（見 SRS_0004）、武將評價演化（見 SRS_0011）。

### 1.3 術語定義

| 術語 | 說明 |
|------|------|
| 家臣團 | 以一位武將為首領的兵力組成單位 |
| 直臣 | 直屬大名、無下屬的家臣 |
| 屬臣 | 直屬宿老/家老、無下屬的家臣 |
| 知行 | 封賜給家臣管理的領地 |
| 一門/譜代/外樣 | 系譜分類，影響派系行為（見 SRS_0010） |

- `0004.封建權謀模組.md` v1.0（忠誠度系統）

## 2. 系統概述

### 2.1 組織架構總覽

```
本家家臣團（大名）
├── 直臣 × N（兵力 1,000 each，無限量）
└── 一級家臣團 × M
    ├── 宿老（兵力 4,000）
    │   ├── 屬臣 × 最多 4（兵力 600 each）
    │   └── 二級家臣團 × 最多 4
    │       ├── 部將（兵力 1,000）→ 組頭 × 最多 4（兵力 200）
    │       └── 侍大將（兵力 600）→ 組頭 × 最多 3（兵力 200）
    └── 家老（兵力 2,000）
        ├── 屬臣 × 最多 3（兵力 600 each）
        └── 二級家臣團 × 最多 4
```

### 2.2 外部相依模組

| 相依模組 | 相依方式 | 用途 |
|---------|---------|------|
| IntrigueSystem | 被呼叫 | 讀取忠誠度決定叛變風險 |
| TerritorySystem | 被呼叫 | 授予知行時同步領地資料 |
| MilitarySystem | 呼叫 | 提供出兵單位（RetainerGroup） |
| RetainerEvalSystem | 被呼叫 | 取得武將評價詞彙 |

## 3. 功能需求（SA 視角）

### 3.1 兵力配置規則

**FR-RET-001**：各階級的兵力上限與任命權限如下表（以此為硬性限制）：

| 階級 | 兵力上限 | 可任命人員 |
|------|---------|-----------|
| 大名 | 5,000 | 宿老/家老 × 1，直臣不限 |
| 宿老 | 4,000 | 屬臣 × 4，二級家臣團 × 4 |
| 家老 | 2,000 | 屬臣 × 3，二級家臣團 × 4 |
| 屬臣 | 600 | 無 |
| 直臣 | 1,000 | 無 |
| 部將 | 1,000 | 組頭 × 4 |
| 侍大將 | 600 | 組頭 × 3 |
| 組頭 | 200 | 無 |

### 3.2 任命邏輯

**FR-RET-002**：大名可任命宿老或家老為一級家臣團首領。每個一級家臣團只能有一位首領。

**FR-RET-003**：宿老/家老可任命部將或侍大將為二級家臣團首領，大名**不得**直接干涉二級家臣團的組成（層級封閉性）。

**FR-RET-004**：任命後，被任命者的直臣/屬臣身分轉換為對應階級，同時喪失被大名直接指揮的能力。

**FR-RET-005**：解除一級家臣團時，其所有成員（含二級家臣團）可選擇「回收為直臣」或「全部解散為浪人」。

### 3.3 系譜分類

**FR-RET-006**：每位家臣在創建時必須指定系譜分類（影響派系行為，見 0010）：

| 分類 | 說明 |
|------|------|
| 一門（CLAN） | 大名親族 |
| 譜代（HEREDITARY） | 世代效忠的舊臣 |
| 外樣（OUTSIDE） | 因戰功或投降歸附的新家臣 |

### 3.4 出兵邏輯

**FR-RET-007**：出兵以「家臣團」為單位，不單獨派遣個人。

**FR-RET-008**：晉升為宿老/家老後，失去對其下層的直接指揮權（必須透過家臣團架構出兵）。

**FR-RET-009**：一個家臣團最大兵力計算範例：

| 首領 | 組合 | 最大兵力 |
|------|------|---------|
| 宿老 | 4 屬臣 + 4 部將各帶 4 組頭 | 13,600 |
| 宿老 | 4 屬臣 + 4 侍大將各帶 3 組頭 | 11,200 |
| 家老 | 3 屬臣 + 4 部將各帶 4 組頭 | 11,000 |
| 家老 | 3 屬臣 + 4 侍大將各帶 3 組頭 | 8,600 |

### 3.5 家臣管理（從核心系統入口）

**FR-RET-010**：透過政務選單「家臣管理」可執行：
- 任免家臣（改變職位或解除任命）
- 評定忠誠（觸發 IntrigueSystem 查看忠誠分級）
- 獎賞家臣（家寶 / 感謝狀 / 賜字 / 改姓），各有對應忠誠度加成

## 4. 非功能需求

| 項目 | 需求 |
|------|------|
| 兵力計算速度 | 全勢力家臣兵力重算 < 0.5 秒 |
| 最大家臣數 | 每勢力 500 人 |
| 資料一致性 | 任命/解除操作須為原子操作（同時更新雙向關聯） |

## 5. 資料模型

```
Retainer (Resource)
├── id: String                    # 唯一識別碼
├── name: String
├── rank: RetainerRank            # 見 enum
├── lineage: LineageType          # 一門/譜代/外樣
├── military_score: int           # 內部數值（0–100），不顯示給玩家
├── strategy_score: int           # 智謀
├── politics_score: int           # 政治
├── charisma_score: int           # 魅力
├── loyalty_value: int            # 內部數值（0–100），玩家見文字分級
├── current_forces: int           # 目前實際兵力（≤ rank 上限）
├── faction_id: String            # 所屬勢力
├── superior_id: String           # 直屬上司（空字串=大名直屬）
├── subordinates: Array[String]   # 直接下屬 ID 列表
└── territory_ids: Array[String]  # 知行領地 ID 列表

RetainerGroup (Resource)
├── id: String
├── leader_id: String             # 家臣團首領 Retainer ID
├── tier: int                     # 0=本家, 1=一級, 2=二級
├── member_ids: Array[String]     # 成員 ID（含子家臣團成員）
└── total_forces: int             # 計算所得總兵力（快取）
```

## 6. GDScript 介面規格（PG 視角）

### 6.1 列舉型別

```gdscript
enum RetainerRank {
    DAIMYO,        # 大名
    SHUKURO,       # 宿老
    KARO,          # 家老
    ZOKUSHIN,      # 屬臣
    JIKISHIN,      # 直臣
    BUSHO,         # 部將
    SAMURAI_TAISH, # 侍大將
    KUMIGASHIRA    # 組頭
}

enum LineageType {
    CLAN,          # 一門
    HEREDITARY,    # 譜代
    OUTSIDE        # 外樣
}

enum RewardType {
    TREASURE,      # 家寶
    COMMENDATION,  # 感謝狀
    NAME_GRANT,    # 賜字
    SURNAME_GRANT  # 改姓
}
```

### 6.2 系統節點

```gdscript
class_name RetainerSystem extends Node

# --- 查詢 ---
func get_retainer(id: String) -> Retainer
func get_all_retainers(faction_id: String) -> Array[Retainer]
func get_direct_subordinates(retainer_id: String) -> Array[Retainer]
func get_retainer_group(leader_id: String) -> RetainerGroup
func get_all_groups(faction_id: String) -> Array[RetainerGroup]

# --- 兵力計算 ---
func get_force_cap(rank: RetainerRank) -> int
    # 回傳對應階級的兵力上限
func calculate_group_total_forces(group: RetainerGroup) -> int
    # 計算家臣團含所有下層的總兵力
func calculate_all_forces(faction_id: String) -> int

# --- 任命操作 ---
func assign_role(retainer_id: String, new_rank: RetainerRank, superior_id: String) -> Dictionary
    # 回傳: {"success": bool, "message": String, "loyalty_delta": int}
func dismiss_retainer(retainer_id: String) -> Dictionary
    # 解除任命，回傳受影響家臣列表
func recall_to_direct(group_id: String) -> bool
    # 解散一級家臣團，成員回收為直臣

# --- 系譜 ---
func set_lineage(retainer_id: String, lineage: LineageType) -> void
func get_lineage(retainer_id: String) -> LineageType

# --- 晉升 ---
func promote_retainer(retainer_id: String, new_rank: RetainerRank) -> Dictionary
    # 回傳: {"success": bool, "message": String, "new_group_created": bool}

# --- 獎賞 ---
func reward_retainer(retainer_id: String, reward_type: RewardType) -> Dictionary
    # 回傳: {"loyalty_delta": int, "message": String}

# --- 驗證 ---
func can_appoint(superior_id: String, new_rank: RetainerRank) -> bool
    # 檢查是否超過任命名額上限
func is_in_same_chain(retainer_a: String, retainer_b: String) -> bool
    # 確認兩人是否在同一指揮鏈（防止越階指揮）
```

## 7. 驗收準則（DoD 對應）

| 驗收項目 | 判定方式 |
|---------|---------|
| 各階級兵力上限正確限制 | 單元測試：嘗試設定超額兵力應回傳 false |
| 任命名額限制正確（宿老最多 4 屬臣等） | 單元測試 |
| 大名無法直接干涉二級家臣團 | 單元測試：呼叫應回傳 false |
| 解散一級家臣團後成員正確回收 | 手動測試 |
| 全勢力兵力重算 < 0.5 秒 | 效能計時 |
