## 1. 引言

### 1.1 目的
本文件定義武將評價模組（RetainerEvalSystem）的軟體需求，涵蓋四面向文字評價系統、評價詞彙定義、演化邏輯與對各系統的數值提供，作為實作與驗收依據。

### 1.2 範圍
RetainerEvalSystem 負責：
- 四面向（軍事 / 智謀 / 政治 / 魅力）的文字評價管理
- 評價等級（E–S 六級）的詞彙表
- 評價演化條件的追蹤與升降
- 對外提供：UI 展示用詞彙 + 各系統計算用內部數值
- 武將特殊評價客製化（特定武將固定成長路徑）

本 SRS **不涵蓋**忠誠度屬性（見 SRS_0004），亦不涵蓋派系思想屬性（見 SRS_0010）。

### 1.3 術語定義

| 術語 | 說明 |
|------|------|
| 評價等級 | E / D / C / B / A / S 六級，由低至高 |
| 評價詞彙 | 對應各等級的文字描述，對玩家顯示 |
| 內部數值 | 用於遊戲計算的 0–100 隱藏數值，玩家不可見 |
| 演化條件 | 觸發評價升降的累積行為條件 |
| 行動記錄 | 武將執行的戰鬥/政務/外交行動的歷史統計 |

## 2. 系統概述

### 2.1 設計哲學
《戰國三代記》不以固定數值顯示武將能力，而是透過武將實際執行的行動自然演化出文字評價。此設計提升角色塑造深度與玩家決策時的情境代入感。

### 2.2 評價體系架構

```
Retainer
├── [顯示層] 評價詞彙（E–S） ← 玩家看到
└── [計算層] 內部數值（0–100） ← 系統使用

RetainerEvalSystem
├── 追蹤行動記錄（ActionLog）
├── 判斷是否達到演化條件
├── 更新評價等級與對應詞彙
└── 對外提供：get_score(dimension) → 內部數值
```

### 2.3 外部相依模組

| 相依模組 | 相依方式 | 用途 |
|---------|---------|------|
| MilitarySystem | 呼叫 RetainerEvalSystem | 戰鬥結果記錄更新軍事評價 |
| TerritorySystem | 呼叫 RetainerEvalSystem | 內政行動更新政治評價 |
| IntelligenceSystem | 呼叫 RetainerEvalSystem | 特務行動更新智謀評價 |
| CouncilSystem | 呼叫 RetainerEvalSystem | 評議說服行動更新魅力評價 |
| IntrigueSystem | 讀取 | 取得內部數值計算忠誠互動 |

## 3. 功能需求（SA 視角）

### 3.1 四面向定義

**FR-EVAL-001**：武將能力評價分為四大面向：

| 面向 | 關聯行為 | 影響系統 |
|------|---------|---------|
| 軍事 | 戰鬥出征、指揮部隊 | MilitarySystem（攻擊力、防禦力、士氣效率） |
| 智謀 | 計謀、特務、外交 | IntelligenceSystem（計謀成功率加成） |
| 政治 | 內政治理、地方施政 | TerritorySystem（軍糧消耗控制、設施加速） |
| 魅力 | 人望、說服、士氣、群眾互動 | MilitarySystem（士氣恢復）、CouncilSystem（說服加成） |

### 3.2 評價等級與詞彙表

**FR-EVAL-002**：四面向各有六個等級，詞彙如下：

#### 軍事評價
| 等級 | 詞彙 | 內部數值範圍 |
|------|------|------------|
| E | 不值一提 | 0–16 |
| D | 稍通軍事 | 17–33 |
| C | 嶄露頭角 | 34–50 |
| B | 頗有勇名 | 51–67 |
| A | 名將之風 | 68–84 |
| S | 無敵猛將 | 85–100 |

#### 智謀評價
| 等級 | 詞彙 | 內部數值範圍 |
|------|------|------------|
| E | 毫無機鋒 | 0–16 |
| D | 略有小計 | 17–33 |
| C | 聰明機智 | 34–50 |
| B | 奇謀之將 | 51–67 |
| A | 謀略過人 | 68–84 |
| S | 智計百出 | 85–100 |

#### 政治評價
| 等級 | 詞彙 | 內部數值範圍 |
|------|------|------------|
| E | 治事無能 | 0–16 |
| D | 可堪使用 | 17–33 |
| C | 能幹之材 | 34–50 |
| B | 為政清明 | 51–67 |
| A | 治國之能臣 | 68–84 |
| S | 天下文臣之宗 | 85–100 |

#### 魅力評價
| 等級 | 詞彙 | 內部數值範圍 |
|------|------|------------|
| E | 不苟言笑 | 0–16 |
| D | 平庸無奇 | 17–33 |
| C | 巧舌如簧 | 34–50 |
| B | 人望漸隆 | 51–67 |
| A | 呼風喚雨 | 68–84 |
| S | 魅力無雙 | 85–100 |

### 3.3 評價演化邏輯

**FR-EVAL-003**：以軍事面向為例，評價升級條件（其他面向類似規則）：

| 目標等級 | 累積參戰次數 | 勝率門檻 |
|---------|-----------|---------|
| D（稍通軍事） | ≥ 2 次 | 任意 |
| C（嶄露頭角） | ≥ 5 次 | ≥ 50% |
| B（頗有勇名） | ≥ 10 次 | ≥ 60% |
| A（名將之風） | ≥ 20 次 | ≥ 70% |
| S（無敵猛將） | ≥ 30 次 | ≥ 80% |

**FR-EVAL-004**：其他面向的演化條件（草案，待平衡確認）：

| 面向 | 累積計數指標 | 升級門檻 |
|------|-----------|---------|
| 智謀 | 情報任務次數 + 計謀成功次數 | 依等級設定成功率門檻 |
| 政治 | 設施升級次數 + 年貢穩定月數 | 依等級設定穩定率門檻 |
| 魅力 | 說服成功次數 + 評議影響他人次數 | 依等級設定成功率門檻 |

**FR-EVAL-005**：評價升級時，內部數值提升到新等級中間值（例：D→C 時，數值設為 42）。

**FR-EVAL-006**：特殊武將可客製化評價進程（固定初始值或特定事件觸發跳升）。

**FR-EVAL-007**：評價劣化機制（v1.1+ 功能，MVP 略過，保留 API 鉤點）：
- 長期未參戰 → 軍事評價下降
- 政務失敗 → 政治評價下降

### 3.4 行動記錄追蹤

**FR-EVAL-008**：系統需追蹤每位武將的以下行動統計：

| 面向 | 追蹤指標 |
|------|---------|
| 軍事 | 參戰次數、勝場次數、敗場次數 |
| 智謀 | 情報任務次數、成功次數 |
| 政治 | 設施升級次數、郡治理次數、軍糧控制效果 |
| 魅力 | 評議說服成功次數、士氣鼓舞次數 |

### 3.5 評價顯示規則

**FR-EVAL-009**：UI 只顯示評價詞彙，不顯示任何數值（0–100 或 E–S 等級標籤不對玩家直接顯示）。

**FR-EVAL-010**：評價變化時顯示敘事描述，例如：
> 「（織田信長）在這場戰役中嶄露頭角，麾下將士對其軍事才幹刮目相看。」

**FR-EVAL-011**：支援多語系詞彙表（中文 / 日文 / 英文），透過外部 JSON 資料表管理。

### 3.6 對各系統提供的數值

**FR-EVAL-012**：各系統可透過 `get_score(retainer_id, dimension)` 取得內部數值（0–100）用於計算：

| 呼叫方 | 取得維度 | 用途 |
|-------|---------|------|
| MilitarySystem | 軍事 | 攻擊力、防禦力修正 |
| MilitarySystem | 魅力 | 士氣恢復量 |
| MilitarySystem | 政治 | 軍糧消耗效率 |
| IntelligenceSystem | 智謀 | 任務成功率加成 |
| TerritorySystem | 政治 | 設施升級加速、恢復加速 |
| CouncilSystem | 魅力 | 說服成功率加成 |

## 5. 資料模型

```
RetainerEval (Resource)           # 掛載於 Retainer
├── military_score: int           # 軍事內部數值 0–100
├── strategy_score: int           # 智謀內部數值
├── politics_score: int           # 政治內部數值
├── charisma_score: int           # 魅力內部數值
├── military_grade: EvalGrade     # 當前軍事評價等級
├── strategy_grade: EvalGrade
├── politics_grade: EvalGrade
├── charisma_grade: EvalGrade
└── action_log: RetainerActionLog

RetainerActionLog (Resource)
├── battles_total: int
├── battles_won: int
├── intel_missions_total: int
├── intel_missions_success: int
├── facility_upgrades: int
├── governance_actions: int
├── persuasion_success: int
└── morale_boost_actions: int

EvalGrade: enum { E, D, C, B, A, S }
```

## 6. GDScript 介面規格（PG 視角）

### 6.1 列舉型別

```gdscript
enum EvalDimension {
    MILITARY,   # 軍事
    STRATEGY,   # 智謀
    POLITICS,   # 政治
    CHARISMA    # 魅力
}

enum EvalGrade {
    E, D, C, B, A, S
}
```

### 6.2 系統節點

```gdscript
class_name RetainerEvalSystem extends Node

# --- 評價查詢（UI 使用）---
func get_grade(retainer_id: String, dimension: EvalDimension) -> EvalGrade
func get_label(retainer_id: String, dimension: EvalDimension) -> String
    # 回傳詞彙，如「頗有勇名」
func get_all_labels(retainer_id: String) -> Dictionary
    # 回傳: {"military": String, "strategy": String, "politics": String, "charisma": String}

# --- 評價查詢（系統計算使用）---
func get_score(retainer_id: String, dimension: EvalDimension) -> int
    # 回傳 0–100 內部數值
func get_score_normalized(retainer_id: String, dimension: EvalDimension) -> float
    # 回傳 0.0–1.0 正規化數值（方便乘以係數）

# --- 行動記錄更新（由各系統呼叫）---
func record_battle(retainer_id: String, won: bool) -> void
    # MilitarySystem 呼叫
func record_intel_mission(retainer_id: String, success: bool) -> void
    # IntelligenceSystem 呼叫
func record_governance(retainer_id: String, action_type: String) -> void
    # TerritorySystem 呼叫
func record_persuasion(retainer_id: String, success: bool) -> void
    # CouncilSystem 呼叫
func record_morale_boost(retainer_id: String) -> void
    # MilitarySystem 呼叫

# --- 演化判斷（每回合或行動後觸發）---
func check_evolution(retainer_id: String, dimension: EvalDimension) -> bool
    # 判斷是否達到升級條件
func evolve_grade(retainer_id: String, dimension: EvalDimension) -> EvalGrade
    # 升級並更新內部數值，回傳新等級
func degrade_grade(retainer_id: String, dimension: EvalDimension) -> EvalGrade
    # 降級（Phase 4+ 使用，MVP 保留鉤點）

# --- 客製化（特殊武將）---
func set_fixed_score(retainer_id: String, dimension: EvalDimension, score: int) -> void
func lock_evolution(retainer_id: String, dimension: EvalDimension, locked: bool) -> void

# --- 敘事描述（評價變化時）---
func get_evolution_narrative(retainer_id: String, dimension: EvalDimension, old_grade: EvalGrade, new_grade: EvalGrade) -> String

# --- 詞彙表（多語系）---
func set_language(lang_code: String) -> void  # "zh_TW", "ja_JP", "en_US"
func reload_vocab_table() -> void
func get_grade_vocab(dimension: EvalDimension, grade: EvalGrade) -> String
```

### 6.3 訊號

```gdscript
signal grade_evolved(retainer_id: String, dimension: EvalDimension, old_grade: EvalGrade, new_grade: EvalGrade)
signal grade_degraded(retainer_id: String, dimension: EvalDimension, old_grade: EvalGrade, new_grade: EvalGrade)
signal action_recorded(retainer_id: String, dimension: EvalDimension)
```

## 7. 驗收準則（DoD 對應）

| 驗收項目 | 判定方式 |
|---------|---------|
| 四面向詞彙表完整（E–S 各 6 個詞彙） | Code Review |

| 軍事演化條件正確（2/5/10/20/30 次） | 單元測試 |
| get_score 回傳正確內部數值 | 單元測試 |
| 詞彙從外部 JSON 讀取（可熱更新） | 手動測試（修改 JSON 後驗證） |
| 特殊武將客製化路徑正確 | 手動測試 |
