## Feedback 1 — 家臣一覽改為條列式，不顯示數值

**需求**：家臣一覽頁面，改為條列式清單，不要顯示數字（忠誠度數值、兵力數字、四面向數值等），只用評價等級（A/B/C/D/S）表示。
**設計決策**：
- 採用 `.retainer-list` / `.retainer-row` / `.rr-*` CSS 類別，取代舊有卡片格（`.retainer-card`）
- 每行顯示：姓名 | 官位 badge | 出身 | 所在城 | 忠誠度文字標籤 | 四面向評價等級 badge
- 數值、長條圖、兵力欄全數移除
**處理**：retainers.html 重寫家臣一覽分頁，CSS style.css 新增條列式樣式規則

## Feedback 2 — 地圖可以拖拉

**需求**：地圖區域支援滑鼠及觸控拖拉（pan），方便檢視各城位置。
**設計決策**：
- `index.html` 地圖內容包裹於 `#map-inner` div（`position:absolute; inset:0`）
- `game.js` 新增 `initMapDrag()`：監聽 `mousedown / mousemove / mouseup / mouseleave` 及等效 `touchstart / touchmove / touchend`
- 以 `e.target.closest('[data-castle]')` 區分「拖拉」與「點擊城節點」，防止誤觸
**處理**：index.html + game.js + CSS `.map-inner` 規則

## Feedback 3 — 家臣團成員管理 UI

**需求**：家臣團管理分頁要能指派家臣入團、從團中移除。
**設計決策**：
- 每個家臣團卡片顯示「指派家臣入團」下拉選單（篩除大名、已在本團的成員）+ 加入按鈕

- 呼叫既有 API：`POST /api/corps/<id>/members`（`action: add/remove`）
**處理**：retainers.html 家臣團管理分頁新增成員操作 UI

## Feedback 7 — 城主（chatelain）與代官（daikan）分離

**需求**：城要有「城主」（軍事管理）、郡要有「代官」（行政管理），兩者角色清楚分開，地圖只顯示城的資訊（城主+兵力），不顯示郡類型標籤。
**設計決策**：
- `castle.chatelain`：城主 retainer_id（軍事）
- `district.daikan`：代官 retainer_id（行政）；舊欄位 `retainer` 棄用
- `data/initial.py`：各城設 `chatelain`，各郡設 `daikan`（部分城郡由不同人擔任，如那古野城：城主=木下藤吉郎・代官=瀧川一益）
- `game/logic.py`：Step 5 忠誠度懲罰改引用 `d["daikan"]`
- `app.py`：`merged_view()` 從郡讀 `daikan`，`/api/castle/<cid>` 回傳 `chatelain` + `daikan`，politics 路由建立 `leader_map`
- `base.html` modal：原 `dm-retainer` 拆分為 `dm-chatelain`（城主・軍事）和 `dm-daikan`（代官・行政）
- 地圖城節點：顯示 `castle-chatelain`（城主名，金色小字）+ 駐守兵力，移除郡類型 tag
**處理**：initial.py / logic.py / app.py / base.html / index.html / game.js / style.css
