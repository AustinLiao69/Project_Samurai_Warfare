## 一、專案背景與目標

《戰國三代記》是單人回合制戰國時代策略模擬遊戲，以織田家為主視角，從永祿三年（1560）桶狹間前夕起，透過政治、軍事、外交、家臣管理等手段實現天下統一。

**MVP 目標**：以 Python Flask Webapp 驗證核心玩法循環，為 Godot 4.x 正式開發提供互動式原型參考。

## 二、核心架構設定

### 2.1 城（Castle）與郡（District）分離原則

- **城（Castle）** = 軍事設施：守備兵力、城等級、攻防計算、地圖位置、鄰接關係
- **郡（District）** = 行政設施：郡類型、設施等級、年貢率、家臣分封、家臣團歸屬、農業開發
- **對應關係**：現階段 1 城對應 1 郡（一城一郡），未來可擴展為 1 城多郡
- **郡類型**：軍事郡（兵營系）/ 商業郡（市集系）/ 農業郡（農村系）

### 2.2 勢力資料

| 勢力 | 城數 | 武將數 | 初始資源 |

| 今川家（NPC） | 3城3郡 | 3人 | 金800・軍糧600 |

## 三、UI 架構

### 3.1 地圖頁（主頁）

- **頂欄（Topbar）**：遊戲標題 | 年號 | 💰金 | 🌾軍糧 | 🏯城數 | ⚔總兵力 | 勢力名
  - **只顯示我方（織田家）資訊**，不顯示今川家數據
- **左欄（Advisor Sidebar）**：
  - 顧問立繪 + 對話框（動態回應滑鼠懸停）
  - 主導航按鈕：政務 / 軍事 / 家臣 / 情報 / 評定
  - **系統設定（整合進主導航下方）**：存檔 / 讀檔 / 重置
  - 結束本月按鈕（紅色行動按鈕）
- **地圖區**：
  - 城堡圖示（可點擊）、連線、省份標籤
  - **地圖可拖拉（pan）**：`map-inner` 整體平移層，支援滑鼠及觸控
  - 城節點顯示：城名 + **城主（chatelain）** + 駐守兵力，**不顯示郡類型**
- **右欄（精簡）**：僅保留「最新消息」事件紀錄，**不顯示我方領地列表**

### 3.2 子頁導航（Subnav）

所有子頁頂端顯示：🗺 地圖 | 📜 政務 | ⚔ 軍事 | 👤 家臣 | 🥷 情報 | 🏛 評定

### 3.3 主要頁面

| 頁面 | 路由 | 功能 |
|------|------|------|
| 地圖 | `/` | 地圖總覽・主導航 |
| 政務 | `/politics` | 領地治理（城+郡管理）・評定快捷 |
| 軍事 | `/military` | 城防徵兵・出兵行動 |
| 家臣 | `/retainers` | 家臣一覽・家臣團管理 |
| 情報 | `/intel` | 佔位符（未來開發） |
| 評定 | `/council` | 本月議題裁決（每月必須） |

## 四、核心遊戲流程（回合循環）

```
每月回合流程：
  1. 玩家在各子頁進行操作（政務/軍事/家臣）
  2. 前往【評定】頁面，裁決本月議題
      └→ 包含：支持 / 反對 / 推遲（三選一）
      └→ 裁決影響家臣忠誠度（支持+3・反對-2・推遲-1）
  3. 點擊【結束本月】
      └→ 若未完成評定裁決 → 阻止並提示「請先前往評定頁面裁決議題」
      └→ 若已裁決 → 執行回合結算
  4. 結算：設施建設完工 / 農兵訓練完成 / 資源計算 / 忠誠度影響 / NPC AI 回合
  5. 結算 log 僅顯示本回合事件（每回合重置，不累計歷史）
  6. 進入下一月，評定裁決狀態清空
```

## 五、功能需求

### UC-001 起始遊戲
- 載入 `INITIAL_STATE`（永祿三年五月）
- 初始城池、郡、武將資料全部設定

### UC-002 地圖頁
- 顯示所有城堡（我方紅色・今川藍色）
- 城堡圖示可點擊 → 開啟城/郡管理 Modal
- 左欄顧問對話、主導航、系統設定、結束本月

### UC-003 存檔/讀檔/重置（系統設定）
- **存檔**：序列化完整 state 至 localStorage
- **讀檔**：從 localStorage 讀回，覆蓋現有進度
- **重置**：回到 INITIAL_STATE
- **入口**：地圖頁左欄主導航下方（系統設定區）

### UC-004 回合結算（評定裁決必填）
- **前提條件**：本月必須已完成評定裁決（支持 / 反對 / 推遲，三選一均可）
- 未裁決時點擊「結束本月」→ 系統提示錯誤並阻止結算
- 結算後事件 log 重置（只顯示本回合事件）
- 結算後評定裁決狀態清空（下月需重新裁決）

### UC-005 評定會議（必填制）
- 8 個議題循環輪換（依 turn index mod 8）
- 每月必須裁決（支持 / 反對 / 推遲），方可結束回合
- 忠誠度影響：同意者 +3・反對者 -2・推遲全體非中立 -1・中立不受影響
- **特殊議題：獻城（ct_02）**
  - 若裁決「屈服：獻出那古野城」→ 那古野城（含那古野郡）割讓予今川家
  - 割讓後：城 faction 更新・郡 corps_id 清空・駐守武將轉移至清洲城
  - 大名本城（清洲城）不可出現在獻城議題中

### UC-010 城/郡 Modal 詳情
- 城屬性：等級、守備兵力、鄰接、位置
- 郡屬性：類型、設施等級/名稱、年貢率、管理家臣、corps 歸屬
- 我方城郡：開放年貢調整、設施升級、城防徵兵
- 家臣團轄地：鎖定直接管理，顯示提示
- 大名本城：可直接管理，顯示 👑 提示、不可分封給家臣團

### UC-011 郡設施升級
- 費用扣除後進入建設期（X 月後完工）
- 軍事郡升級 → 城等級同步提升
- 完工後 log 記錄（當回合）

### UC-012 年貢率調整
- 範圍：10%—60%（步距 5%）
- 超過 30% → 每月忠誠度下降（比例計算）
- 即時生效，即時更新收入預覽

### UC-020 城兵徵召（即時）
- 費用：50 金 / 300 人（即時補充到城）

### UC-021 農民兵徵集（延遲）
- 費用：30 金 / 500 人（1 月後完成）
- 郡 farmer_pending/farmer_incoming 欄位追蹤

### UC-022 野戰
- 出兵武將選擇 → 模擬對戰 → 攻方勝利則奪城

### UC-023 圍城
- 每回合消耗守方軍糧，軍糧歸零時城池陷落

### UC-030 家臣團管理
- 成立：首領（宿老/家老）+ 名稱 + 初始轄城（郡）
- 轄地：可追加分配城郡（大名本城禁止）
- 解散：歸還所有轄地管理權
- 限制：轄地城郡不可直接由玩家管理（Modal 鎖定，鎖定項目：年貢・郡政升級・城防徵兵・出兵）
- **成員管理**：家臣團管理分頁可指派家臣入團 / 移除成員，呼叫 `/api/corps/<id>/members`

### UC-031 家臣一覽（條列式）
- 顯示格式：條列式（非卡片 Grid）
- 顯示欄位：姓名・官位 badge・出身・所在城・忠誠度文字標籤・四面向評價等級（A/B/C/D/S）
- **不顯示數值**（忠誠度數字、兵力數字、四面向數值、忠誠度長條圖均隱藏）
- 可依派閥（織田家 / 今川家）切換顯示

### UC-032 城主（Chatelain）與代官（Daikan）分離
- **城主（Chatelain）**：城的軍事管理人，保存於 `castle.chatelain`，顯示於地圖節點及城 Modal
- **代官（Daikan）**：郡的行政管理人，保存於 `district.daikan`，顯示於政務頁及郡 Modal
- 同一武將可同時擔任某城城主及其轄郡代官（如清洲城初期柴田勝家身兼兩職）
- 忠誠度懲罰邏輯（`logic.py`）引用 `daikan` 而非舊的 `retainer` 欄位

## 六、資料模型

### Castle（城）— 軍事設施
```
id, name, faction, level, garrison, max_garrison,
province, is_capital, is_daimyo_home, x, y,
adjacent[],      # 鄰接城 id 列表
district_id,     # 1:1 對應的郡 id
chatelain        # 城主 retainer_id（軍事管理人）
```

### District（郡）— 行政設施
```
id, name, castle_id,
type（軍事郡 / 商業郡 / 農業郡）,
facility_name, facility_level, facility_max,
building（bool）, building_turns（int）,
nengu_rate（10–60）,
daikan（retainer_id）,   # 代官（行政管理人）；舊欄位 retainer 已棄用
corps_id（corps_id | None）,
farmer_pending（int）, farmer_incoming（int）
```

### Retainer（武將）
```
id, name, faction,
rank（大名 / 宿老 / 家老 / 部將 / 組頭）,
lineage（譜代 / 外樣 / 一門）,
loyalty（0–100）, loyalty_label,
forces, location（castle_id）, corps_id,
mil（武力）, str_（統率）, pol（智略）, cha（魅力）,
eval（A/B/C/D/S 評價）
```

### Corps（家臣團）
```
id, name,
leader（retainer_id），必須宿老或家老
members[]（retainer_id）,
territories[]（castle_id）
```

### GameState（完整狀態）
```
date: {year, month}
player_faction: "oda"
turn: int
log: [str]         # 本回合事件（每回合重置）
council_decision:  # {choice_id, label} | None（None=本月尚未裁決）
factions: {oda: {...}, imagawa: {...}}
castles: [Castle]
districts: [District]
retainers: [Retainer]
corps: [Corps]
```

評定裁決影響：
- 與武將立場相同 → +3
- 與武將立場相反 → -2
- 推遲（delay） → 非中立武將 -1
- 中立武將 → 任何裁決均不影響

## 十、API 規格

| 路由 | 方法 | 功能 |
|------|------|------|
| `/api/castle/<cid>` | GET | 取得城+郡合併資訊 |
| `/api/castle/<cid>/upgrade` | POST | 郡設施升級 |
| `/api/castle/<cid>/nengu` | POST | 年貢率設定 |
| `/api/mobilize` | POST | 城防徵兵 |
| `/api/battle` | POST | 出兵（野戰/圍城） |
| `/api/end-turn` | POST | 結束回合（需先裁決） |
| `/api/council-decision` | POST | 評定裁決 |
| `/api/corps` | GET/POST | 家臣團查詢/建立 |
| `/api/corps/<id>` | DELETE | 解散家臣團 |
| `/api/corps/<id>/members` | POST | 成員管理 |
| `/api/corps/<id>/territories` | POST | 轄地管理 |
| `/api/load-state` | POST | 讀取存檔 |
| `/api/reset` | POST | 重置遊戲 |
| `/api/state` | GET | 取得完整狀態 |
