## 2. 專案背景與動機 (Background)

《戰國三代記》的誕生源自對現有戰國策略遊戲的觀察。市場上雖有《信長之野望》系列、《全軍破敵：三國》等作品，但在以下面向仍存在明顯空白：

- **家臣團內部政治**的深度模擬不足
- **封建階級體系**（大名→宿老→部將→組頭）缺乏動態互動
- **忠誠度、派系鬥爭、叛變風險**欠缺真實感
- **三層級領地管理**（令制國→城→郡）的策略深度不夠

本專案以填補此市場缺口為核心目標，打造一款讓玩家真正感受「戰國領主」政治壓力與統治藝術的模擬遊戲。

---

## 3. 產品願景 (Product Vision)

> **讓玩家感受到，統一天下最難的不是戰場，而是自家的評定座席。**

《戰國三代記》的獨特價值在於：每一個決策都會在家臣團內部引發政治漣漪。玩家必須同時應對外部戰局與內部權謀，才能在亂世中傳承三代霸業。

## 4. 目標市場與玩家輪廓 (Target Audience)

### 主要客群
| 分類 | 說明 |
|------|------|
| **歷史迷** | 喜愛日本戰國時代歷史，熟悉信長、秀吉、家康等人物 |
| **策略遊戲玩家** | 習慣 4X 策略、Grand Strategy 類型 |
| **政治模擬愛好者** | 偏好《十字軍之王》、《維多利亞》式的社會關係模擬 |

### 次要客群
- 輕度策略玩家（透過 UX 設計降低進入門檻）
- 日本文化愛好者

### 競品定位
| 競品 | 我們的差異化 |
|------|-------------|
| 《信長之野望》系列 | 更深的家臣政治互動、更真實的忠誠度動態 |
| 《全軍破敵》 | 更聚焦封建社會結構，減少即時戰鬥複雜度 |
| 《十字軍之王 3》 | 聚焦日本戰國特色，玩法更直覺 |

## 5. 專案目標 (Project Objectives)

### 5.1 業務目標
- 在 Steam / itch.io 完成正式發布
- Early Access 階段達成 1,000 個售出授權
- 玩家好評率目標 ≥ 75%（Steam 正評率）

### 5.2 產品目標

| 優先序 | 目標 |
|--------|------|
| P0 | 完成 MVP（Phase 1-3）：核心系統、領地管理、基礎軍事 |
| P1 | 完成完整版（Phase 4-6）：權謀、情報、高級軍事 |
| P2 | 上線後優化迭代（Phase 7）：平衡、Bug 修復、UI 精修 |

### 5.3 品質目標
- 回合處理時間 < 3 秒
- 介面響應時間 < 0.5 秒
- 記憶體使用 < 1 GB
- 連續遊玩 4 小時無崩潰
- 存檔成功率 > 99.9%

## 6. 專案範圍 (Scope)

### 範圍內 (In Scope)
- 7 大遊戲系統模組開發（核心、領地、家臣、權謀、軍事、評議、情報）
- 單人遊戲模式
- PC 三平台支援（Windows / macOS / Linux）
- 基礎存讀檔系統（10 個存檔槽）
- 自創武將系統（P2 優先級）
- 成就系統（P2 優先級）

### 範圍外 (Out of Scope)
- 多人連線 / 多人對戰模式
- 行動平台（iOS / Android）
- 主機平台（PS5 / Xbox / Switch）
- 官方 Mod 工具支援
- 完整語音配音（文字版本優先）

## 9. 假設與限制 (Assumptions & Constraints)

### 假設
- 開發工具：Godot 4.x 穩定版本持續維護
- 開發語言：GDScript（不引入 C# 或外部插件）
- 開發資源：小型開發團隊（1-3 人）或 solo developer
- 美術資源：初期使用 AI 生成角色肖像，後期精修

### 限制
- 預算限制：獨立遊戲規模，無大型行銷預算
- 時程限制：30 週為預設開發週期，不含後續 DLC
- 技術限制：Godot 4.x 的跨平台兼容性邊界需持續驗證
