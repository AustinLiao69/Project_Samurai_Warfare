## 2. MVP 總覽 (Minimum Viable Product)

> **MVP 定義**：Phase 1 ＋ Phase 2 ＋ Phase 3（Week 3-13，共 11 週開發）  
> **目標**：讓玩家可以選一個勢力開始遊戲、治理封地、發動戰役，並將進度存起來。

### MVP 功能範圍

| 功能類別 | 包含的功能 | 對應 Phase |
|----------|-----------|-----------|
| **遊戲入口** | 新遊戲（選劇本 / 選勢力）、讀取存檔、自動存檔 | Phase 1 |
| **回合系統** | 月份推進、季節變化、年度結算 | Phase 1 |
| **存讀檔** | 10 槽手動存檔、自動存檔、快速存檔 | Phase 1 |
| **領地管理** | 三層級領地瀏覽、設施建設與升級、年貢率調整 | Phase 2 |
| **資源管理** | 每月金錢 / 兵糧 / 人口結算 | Phase 2 |
| **兵力系統** | 城兵（立即可用）/ 農兵（需 1 個月集結） | Phase 3 |
| **軍事行動** | 出征指令、野戰結算、圍城戰、佔領轉移 | Phase 3 |

### MVP 明確排除的功能

| 排除功能 | 原因 | 列入 |
|----------|------|------|
| 忠誠度動態 / 派系鬥爭 | 複雜度高，列為完整版 | Phase 4 |
| 評定會議 | 依賴派系系統 | Phase 4 |
| 忍者 / 情報系統 | 獨立模組，不影響 MVP 核心體驗 | Phase 5 |
| 分層攻城（本丸 / 一之丸） | 軍事進階功能 | Phase 6 |
| 自創武將 / 成就系統 | 非核心遊戲體驗 | 發布後 P2 |

### MVP 里程碑（M3）通過條件

```
✅ 可以啟動遊戲並選擇勢力進入
✅ 回合可以連續推進（至少 12 個月）
✅ 存檔後讀取，所有數據完整還原
✅ 可以管理至少 3 個郡的設施與年貢
✅ 可以對鄰接郡發動攻擊並解決戰鬥
✅ 連續遊玩 1 小時無崩潰
```

## 4. Phase 1：核心系統 (Week 3-6)

### 目標
建立遊戲的骨架：可以啟動、儲存、讀取，並進行基本回合循環。

### Sprint 計畫

#### Sprint 1.1（Week 3-4）：遊戲基礎架構
| 任務 | 預估時間 | 模組 |
|------|----------|------|
| 專案初始化（Godot 4 設定、目錄結構） | 0.5d | 核心 |
| MainGame 控制器 + GameStateManager | 1d | 核心 |
| SaveSystem（存 / 讀 10 槽） | 1.5d | 核心 |
| 主選單 UI（新遊戲 / 讀取 / 設定） | 1d | UI |
| 基礎資料結構（Faction / Retainer / Territory Resource） | 2d | 資料層 |

#### Sprint 1.2（Week 5-6）：回合系統 + 基礎指令
| 任務 | 預估時間 | 模組 |
|------|----------|------|
| 回合推進引擎（月份 / 季節 / 年度） | 2d | 核心 |
| 指令處理系統（GameCommand 架構） | 1.5d | 核心 |
| 初始場景設定器（劇本資料載入） | 1d | 核心 |
| 基礎事件通知系統 | 1d | 核心 |
| Phase 1 整合測試 | 1d | QA |

### Phase 1 完成標準（DoD）
- [ ] 遊戲可以啟動並顯示主選單
- [ ] 新遊戲可以選擇劇本與勢力後進入遊戲
- [ ] 回合可以推進（月份遞增正確）
- [ ] 存檔可以儲存並成功讀取（驗證 10 槽均可用）
- [ ] 無啟動崩潰

### 里程碑 M1 - 核心系統可運行
> **通過條件**：能完成「開始遊戲 → 進行 12 個回合 → 存檔 → 讀取 → 繼續」完整流程
