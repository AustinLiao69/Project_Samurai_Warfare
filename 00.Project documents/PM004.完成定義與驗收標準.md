## 1. 完成定義（Definition of Done）框架

DoD 分為三個層級，每個層級都必須滿足，才算「真正完成」：

```
Level 1：任務完成（Task Done）
  ↓
Level 2：功能完成（Feature Done）
  ↓
Level 3：里程碑完成（Milestone Done）
```

---

## 2. Level 1：任務完成標準（適用每個開發任務）

任何一個開發任務，必須同時滿足以下全部條件才算完成：

| # | 完成條件 | 驗證方式 |
|---|----------|----------|
| 1 | 功能實作完畢，符合設計文件規格 | Code Review / 設計文件比對 |
| 2 | 程式碼可以正常執行，無 runtime 錯誤 | 手動測試執行 |
| 3 | 函數有適當的命名（遵循 GDScript 命名規範） | Code Review |
| 4 | 關鍵邏輯有內嵌說明（複雜算法須加 comment） | Code Review |
| 5 | 不引入新的 Godot 警告（Warning-free） | Godot 編輯器確認 |
| 6 | 已推送至 Git（commit message 清楚描述變更） | Git log 確認 |
| 7 | 不破壞既有功能（Regression-free） | 手動跑現有流程 |

## 3. Level 2：功能完成標準（適用每個 User Story / Feature）

| # | 完成條件 | 驗證方式 |
|---|----------|----------|
| 1 | 滿足對應 User Story 的所有驗收標準 | PM 驗收 |
| 2 | 功能在三種情境下均測試過：正常路徑 / 邊界條件 / 錯誤輸入 | QA 測試清單 |

| 4 | 效能在可接受範圍內（無明顯卡頓） | 主觀測試 |
| 5 | 與相關模組的整合測試通過 | 整合測試清單 |
| 6 | 功能更新已同步更新相關設計文件（如有變更） | 文件審查 |

## 5. 各模組功能驗收標準（Acceptance Criteria）

### 5.1 核心系統（CoreSystem）

#### AC-CORE-001：回合推進
```
Given 遊戲在第 X 月
When  玩家執行「結束回合」
Then  月份變為 X+1
And   如果 X=12，年份 +1 且月份歸 1
And   季節根據月份正確更新（春/夏/秋/冬）
And   年貢收取在對應月份自動觸發
```

#### AC-CORE-002：存檔
```
Given 玩家選擇存入第 3 槽
When  存檔完成
Then  第 3 槽顯示當前勢力名稱、月份、年份
And   下次讀取第 3 槽時，所有數據與存檔時完全一致
And   其他存檔槽不受影響
```

### 5.2 家臣團系統（RetainerSystem）

#### AC-RETAINER-001：忠誠度計算
```
Given 家臣的忠誠度為 65（表面恭敬）
When  玩家對該家臣執行「賞賜金銀」（消耗 100 金）
Then  忠誠度增加 5-15 點（依家臣個性浮動）
And   如果忠誠度超過 75，顯示「忠心耿耿」稱號
And   如果家臣屬於「譜代」身分，增加量額外 × 1.2
```

#### AC-RETAINER-002：晉升
```
Given 家臣目前階級為「部將」
When  玩家選擇「拔擢」該家臣至「家老」
Then  家臣階級更新為「家老」
And   其他家老對此晉升事件有反應（忠誠度微調）
And   新家老的知行（領地分配上限）依新階級更新
And   若無空缺，系統提示「家老人數已達上限」
```

### 5.3 領國系統（TerritorySystem）

#### AC-TERRITORY-001：設施升級
```
Given 農業郡「山城郡」的農田等級為 2
And   玩家擁有 200 金、50 兵糧
When  玩家選擇升級農田至等級 3（需 150 金）
Then  金錢扣減 150，剩餘 50 金
And   升級需 1 個月，農田顯示「建設中」
And   下個月農田等級更新為 3
And   該郡的年貢產出依新等級計算
```

### 5.4 軍事系統（MilitarySystem）

#### AC-MILITARY-001：野戰結算
```
Given 攻方兵力 3000，統率武將統率值 80
And   守方兵力 1500，統率武將統率值 60
When  發生野戰
Then  攻方勝利概率高（依公式計算）
And   雙方均有兵力損失（損失量與戰鬥強度成正比）
And   敗方撤退，不喪失主帥
And   若守方武將統率 ≥ 85，可能觸發「奇襲」事件
```

### 5.5 情報系統（IntelligenceSystem）

#### AC-INTEL-001：忍者刺探
```
Given 派遣一名忍者前往敵勢力「今川」刺探
When  任務成功（成功率 = 忍者屬性 / 敵方警戒）
Then  解鎖今川勢力的部分情報（兵力 / 忠誠度之一）
And   忍者在下一月才能執行新任務
When  任務失敗
Then  忍者被捕，從家臣名單移除
And   對「今川」的外交關係惡化
```

## 6. Bug 優先序定義

| 等級 | 定義 | 例子 | 目標修復時間 |
|------|------|------|-------------|
| **P0 - 阻塞** | 核心流程崩潰或資料損毀 | 存檔後讀取失敗、遊戲啟動崩潰 | 24 小時內 |
| **P1 - 嚴重** | 重要功能無法使用，但有繞道方法 | 忠誠度不更新、攻城結果錯誤 | 3 天內 |
| **P2 - 一般** | 功能基本可用，但有異常 | UI 顯示錯亂、數字顯示格式錯誤 | 下個 Sprint |
| **P3 - 輕微** | 美觀或體驗問題 | 字型大小不一致、動畫不流暢 | Phase 7 優化期 |
