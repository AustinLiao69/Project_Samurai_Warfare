## 1. 引言

### 1.1 目的
本文件定義派系思想模組（FactionSystem）的軟體需求，涵蓋武將六維度派系屬性的定義、行為傾向、遊戲效果與派系互動，作為實作與驗收依據。

### 1.2 範圍
FactionSystem 負責：
- 每位武將的六維度派系屬性管理
- 各派系對行為的傾向（影響評議表態、策反反應等）
- 派系間的衝突與合作關係
- 派系立場對忠誠度變化的修正
- 繼承人派系的動態形成

本 SRS **不涵蓋**忠誠度底層數值（見 SRS_0004），亦不涵蓋評議表態的完整邏輯（見 SRS_0007）。

### 1.3 術語定義

| 術語 | 說明 |
|------|------|
| 派系屬性 | 武將在六個維度上的立場分類 |
| 傾向 | 某派系在特定議題上的預設支持/反對傾向 |
| 派系衝突 | 兩個立場相反的派系之間的矛盾（影響互動事件） |

### 1.4 參考文件
- `0010.派系思想模組.md` v0.3
- `0004.封建權謀模組.md` v1.0（派系形成）
- `0007.評議模組.md` v1.1（評議表態）

## 2. 系統概述

### 2.1 六維度架構
每位武將同時持有六個派系屬性，每個維度只能選一個值（排他性）：

| 維度 | 選項 |
|------|------|
| 1. 出身分派 | 譜代派 / 外樣派 / 直臣派 |
| 2. 政務分派 | 武功派 / 文治派 |
| 3. 家臣團首領派 | 依現有宿老/家老動態生成（如「柴田派」「羽柴派」） |
| 4. 繼承人派 | 世子派 / 二子派 / 養子派 |
| 5. 朝廷幕府態度 | 勤王派 / 幕府派 / 割據派 |
| 6. 年資派 | 老臣派 / 新進派 |

### 2.2 外部相依模組

| 相依模組 | 相依方式 | 用途 |
|---------|---------|------|
| CouncilSystem | 被呼叫 | 讀取派系立場決定評議表態傾向 |
| IntrigueSystem | 被呼叫 | 讀取派系決定忠誠度修正係數 |
| RetainerSystem | 讀取 | 建立武將與家臣團首領派的對應 |
| CoreSystem | 被呼叫 | 繼承人任命時動態更新繼承人派 |

## 3. 功能需求（SA 視角）

### 3.1 維度一：出身分派

**FR-FAC-001**：出身分派遊戲效果：

| 派系 | 遊戲效果 | 衝突對象 |
|------|---------|---------|
| 譜代派 | 忠誠穩定高，保守抗拒改革；評議傾向反對改革政策 | 外樣派、直臣派 |
| 外樣派 | 野心大，渴望晉升；忠誠較不穩定，易被策反；評議立場分歧 | 譜代派 |
| 直臣派 | 對大名本人忠誠高；被視為寵臣受譜代排擠；大名死後忠誠快速下降 | 譜代派 |

### 3.2 維度二：政務分派

**FR-FAC-002**：政務分派遊戲效果：

| 派系 | 遊戲效果 | 評議傾向 |
|------|---------|---------|
| 武功派 | 提升軍事效率與士氣；內政效率低，民心可能下降 | 贊成軍事行動；反對和談 |
| 文治派 | 提升內政效率與民心；軍事決策中常與武功派衝突 | 反對戰爭；贊成改革/制度建設 |

### 3.3 維度三：家臣團首領派

**FR-FAC-003**：此維度動態生成，以現有宿老/家老的名字命名（如「羽柴派」「柴田派」）。

**FR-FAC-004**：武將加入家臣團後，預設傾向該首領所屬派系（可因後續事件改變）。

**FR-FAC-005**：首領死亡或失去地位後，其派系成員重新分配傾向（隨機或依忠誠度）。

**FR-FAC-006**：大名若偏袒某個首領派，其他首領派的成員忠誠度下降。

### 3.4 維度四：繼承人派

**FR-FAC-007**：大名子孫元服或收養養子後，系統自動生成對應繼承人派別。

**FR-FAC-008**：各繼承人派特性：

| 派系 | 支持者構成 | 遊戲效果 |
|------|----------|---------|
| 世子派 | 老臣、譜代為主 | 穩定，世子資質平庸時爭議較多 |
| 二子派 | 新進派、外樣為主 | 可能引發繼承內亂 |
| 養子派 | 與外部勢力連動的家臣 | 引入外部勢力干涉，容易分裂 |

### 3.5 維度五：朝廷幕府態度

**FR-FAC-009**：三種朝廷/幕府立場效果：

| 派系 | 遊戲效果 | 評議傾向 |
|------|---------|---------|
| 勤王派 | 容易獲得朝廷支持與合法性；強烈反感「割據」行為 | 贊成奉朝廷詔命的行動 |
| 幕府派 | 容易獲得幕府援助；與勤王派或割據派衝突 | 贊成配合幕府的行動 |
| 割據派 | 強化內政獨立性；易遭朝廷、幕府圍剿 | 反對任何需服從中央的行動 |

### 3.6 維度六：年資派

**FR-FAC-010**：年資派效果：

| 派系 | 遊戲效果 | 評議傾向 |
|------|---------|---------|
| 老臣派 | 忠誠穩定，維護秩序；抗拒改革 | 支持保守政策、傳統功勞排序 |
| 新進派 | 帶來活力；容易不滿現狀，反叛風險較高 | 支持改革、積極擴張 |

### 3.7 派系衝突矩陣

**FR-FAC-011**：主要派系衝突對（衝突會增加互動事件觸發機率）：

| 衝突對 | 衝突類型 |
|--------|---------|
| 譜代派 vs 外樣派 | 地位之爭（評議表態對立） |
| 武功派 vs 文治派 | 政策方向對立 |
| 世子派 vs 二子派 / 養子派 | 繼承權之爭 |
| 勤王派 vs 割據派 | 中央/獨立立場對立 |
| 老臣派 vs 新進派 | 資歷與機會競爭 |

### 3.8 派系立場對忠誠度的修正

**FR-FAC-012**：當評議裁決方向與武將派系立場一致時，額外忠誠加成 +5；若完全相反，額外忠誠懲罰 -5（疊加在 SRS_0007 的基礎 delta 上）。

## 4. 非功能需求

| 項目 | 需求 |
|------|------|
| 派系初始化 | 遊戲開始時依 PM008 資料自動為每位武將設定派系 |

## 5. 資料模型

```
RetainerFactionProfile (Resource)   # 掛載於 Retainer
├── origin_faction: OriginFaction   # 出身分派
├── politics_faction: PoliticsFaction  # 政務分派
├── leader_faction_id: String       # 家臣團首領派（動態，對應 Retainer ID）
├── heir_faction: HeirFaction       # 繼承人派
├── court_faction: CourtFaction     # 朝廷幕府態度
├── seniority_faction: SeniorityFaction  # 年資派
└── conflict_scores: Dictionary     # {retainer_id: float}（與各武將的衝突指數）

LeaderFaction (Resource)            # 家臣團首領派（動態生成）
├── id: String
├── leader_retainer_id: String
├── name: String                    # 如「柴田派」
└── member_ids: Array[String]
```

## 6. GDScript 介面規格（PG 視角）

### 6.1 列舉型別

```gdscript
enum OriginFaction {
    HEREDITARY,  # 譜代派
    OUTSIDE,     # 外樣派
    DIRECT       # 直臣派
}

enum PoliticsFaction {
    MILITARY_FIRST, # 武功派
    CIVIL_FIRST     # 文治派
}

enum HeirFaction {
    HEIR_APPARENT,  # 世子派
    SECOND_SON,     # 二子派
    ADOPTED         # 養子派
}

enum CourtFaction {
    ROYALIST,   # 勤王派
    SHOGUNATE,  # 幕府派
    ISOLATIONIST# 割據派
}

enum SeniorityFaction {
    VETERAN,    # 老臣派
    RISING      # 新進派
}
```

### 6.2 系統節點

```gdscript
class_name FactionSystem extends Node

# --- 派系查詢 ---
func get_faction_profile(retainer_id: String) -> RetainerFactionProfile
func get_origin_faction(retainer_id: String) -> OriginFaction
func get_politics_faction(retainer_id: String) -> PoliticsFaction
func get_heir_faction(retainer_id: String) -> HeirFaction
func get_court_faction(retainer_id: String) -> CourtFaction
func get_seniority_faction(retainer_id: String) -> SeniorityFaction
func get_leader_faction(retainer_id: String) -> LeaderFaction

# --- 派系初始化 ---
func initialize_faction_profile(retainer_id: String, profile_data: Dictionary) -> void
func assign_random_faction(retainer_id: String, exclude_factions: Array) -> void

# --- 評議傾向計算 ---
func get_council_bias(retainer_id: String, council_type: CouncilType, topic: String) -> int
    # 回傳傾向值: -2=強烈反對, -1=傾向反對, 0=中立, +1=傾向贊成, +2=強烈贊成

# --- 忠誠度修正 ---
func get_loyalty_modifier(retainer_id: String, council_decision_direction: int) -> int
    # 回傳忠誠 delta 修正: +5（一致）/ 0（無關）/ -5（相反）

# --- 衝突關係 ---
func get_conflict_score(retainer_a: String, retainer_b: String) -> float
    # 回傳 0.0–1.0，越高衝突越強
func are_in_conflict(retainer_a: String, retainer_b: String) -> bool

# --- 家臣團首領派動態更新 ---
func create_leader_faction(leader_retainer_id: String) -> LeaderFaction
func dissolve_leader_faction(leader_retainer_id: String) -> void
func reassign_members_after_dissolution(dissolved_faction_id: String) -> void

# --- 繼承人派 ---
func generate_heir_factions(heir_retainer_ids: Dictionary) -> void
    # heir_retainer_ids: {"heir_apparent": String, "second_son": String, "adopted": String}
func get_heir_faction_members(heir_faction: HeirFaction) -> Array[String]

# --- 派系影響查詢 ---
func get_military_efficiency_modifier(retainer_id: String) -> float  # 武功派 > 文治派
func get_civil_efficiency_modifier(retainer_id: String) -> float     # 文治派 > 武功派
func get_court_support_modifier(retainer_id: String) -> float        # 勤王派朝廷加成
```
