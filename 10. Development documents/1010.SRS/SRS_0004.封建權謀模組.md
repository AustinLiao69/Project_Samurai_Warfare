## 1. 引言

### 1.1 目的
本文件定義封建權謀模組（IntrigueSystem）的軟體需求，聚焦於忠誠度系統、派系機制與叛亂風險，作為程式實作與驗收依據。

### 1.2 範圍
IntrigueSystem 負責：
- 家臣忠誠度的內部數值管理與文字分級顯示
- 忍者對家臣的監控（與 IntelligenceSystem 協作）
- 派系（支持不同繼承人的集團）的形成與互動
- 叛亂事件觸發

本 SRS **不涵蓋**忍者的派遣與任務邏輯（見 SRS_0008），亦不涵蓋評議對忠誠度的影響（見 SRS_0007）。

### 1.3 術語定義

| 術語 | 說明 |
|------|------|
| 忠誠度 | 家臣對主君信任的唯一屬性，以文字分級對外顯示 |
| 文字分級 | 玩家看到的忠誠描述詞（六級），內部儲存 0–100 數值 |
| 派系 | 以擁立特定繼承人為核心的家臣集團 |
| 過期情報 | 未派忍者監控的家臣，其忠誠分級可能停留在舊值 |

## 2. 系統概述

### 2.1 模組定位
IntrigueSystem 是忠誠度的「真相持有者」（Single Source of Truth）。其他系統（評議、情報、戰後處置）寫入忠誠度時，必須透過 IntrigueSystem 的 API，不得直接修改 Retainer 的 `loyalty_value`。

### 2.2 外部相依模組

| 相依模組 | 相依方式 | 用途 |
|---------|---------|------|
| IntelligenceSystem | 呼叫 | 查詢忍者對某家臣的探查結果 |
| CouncilSystem | 被呼叫 | 評議後忠誠度變化寫入 |
| PostWarSystem | 被呼叫 | 戰後收編時初始忠誠度設定 |
| RetainerSystem | 讀取 | 取得家臣列表 |
| FactionSystem | 雙向 | 派系影響忠誠；忠誠影響派系勢力 |

## 3. 功能需求（SA 視角）

### 3.1 忠誠度系統

**FR-INT-001**：每位家臣持有唯一忠誠度屬性（`loyalty_value`），內部為整數 0–100，對玩家僅顯示文字分級。

**FR-INT-002**：忠誠度文字分級定義：

| 文字分級 | 顯示詞 | 內部數值範圍 |
|---------|--------|-------------|
| ABSOLUTE | 臣服如山 | 91–100 |
| HIGH | 忠心耿耿 | 76–90 |
| MODERATE | 表面恭敬 | 61–75 |
| UNCERTAIN | 意圖未明 | 41–60 |
| LOW | 暗懷異志 | 21–40 |
| TRAITOR | 潛在叛徒 | 0–20 |

**FR-INT-003**：玩家**無法**直接看到 0–100 的數值。UI 僅顯示文字分級與對應的情境描述句（例：「高坂昌信面色陰沉，沉默不語。」）。

**FR-INT-004**：忠誠度會因以下事件變動，各事件的 delta 範圍：

| 事件 | 忠誠 delta |
|------|-----------|
| 評議立場與裁決一致 | +5 ~ +15 |
| 評議立場與裁決相反 | -5 ~ -20 |
| 獲得封賞、官職 | +10 ~ +25 |
| 地位被削弱 | -15 ~ -30 |
| 連續多次被忽視 | 額外 -10 ~ -20 |
| 年貢過高（領國系統） | -1 ~ -5 / 回合 |
| 戰後不滿（戰後處置） | 依公式計算 |

**FR-INT-005**：忠誠度邊界夾緊在 [0, 100]，不允許溢出。

### 3.2 情報遮蔽機制（資訊不完全）

**FR-INT-006**：玩家看到的忠誠分級來源有兩種：

| 來源 | 條件 |
|------|------|
| 即時分級 | 該回合有忍者成功探查過此家臣 |
| 過期分級 | 未探查，顯示上次成功探查時的分級；若從未探查，依初始設定顯示 |

**FR-INT-007**：若家臣實際忠誠度已跌入「暗懷異志」或「潛在叛徒」，但玩家未探查，顯示仍保持舊分級，可能導致玩家誤判。

**FR-INT-008**：高階忍者（等級 ≥ 3）才能探查「暗懷異志」與「潛在叛徒」等低忠誠分級，普通忍者最多揭露「意圖未明」。

### 3.3 派系系統

**FR-INT-009**：大名子孫元服或收養養子後，系統自動生成「擁立派系」，各家臣依個人立場分配初始傾向。

**FR-INT-010**：派系行為包括：
- 爭奪繼承權（影響特別評議的表態）
- 策反他人家臣（降低目標忠誠度）
- 暗殺目標（觸發高風險事件，MVP 暫不實作）
- 支持換主（擁立新大名，MVP 暫不實作）

**FR-INT-011**：大名偏袒某派系（連續多次評議支持同一派）會導致其他派系集體忠誠度下降。

### 3.4 叛亂觸發

**FR-INT-012**：當家臣忠誠度連續 3 回合處於「潛在叛徒」（0–20），且玩家未採取任何應對措施，系統觸發叛變事件（呼叫 CoreSystem.trigger_event）。

**FR-INT-013**：叛變事件的嚴重程度與叛變家臣的階級成正比（宿老叛變 > 組頭叛變）。

## 4. 非功能需求

| 項目 | 需求 |
|------|------|
| 忠誠度更新頻率 | 每回合結算一次批次更新 |
| 並發安全 | 多事件同回合修改同一家臣忠誠度時，delta 累加後再套用 |
| 效能 | 500 名家臣全體忠誠度結算 < 0.1 秒 |

## 5. 資料模型

```
LoyaltyRecord (儲存於 Retainer)
├── loyalty_value: int           # 內部真實數值 0–100
├── last_surveyed_turn: int      # 最後一次忍者成功探查的回合
├── displayed_grade: LoyaltyGrade # 目前對玩家顯示的分級（可能過期）
└── consecutive_low_turns: int   # 連續低忠誠（0–20）回合數

FactionGroup (Resource)
├── id: String
├── name: String                 # 如「世子派」「次子派」
├── target_heir_id: String       # 擁立的繼承人 ID
├── member_ids: Array[String]    # 成員家臣 ID 列表
└── influence: float             # 派系勢力值（0.0–1.0）
```

## 6. GDScript 介面規格（PG 視角）

### 6.1 列舉型別

```gdscript
enum LoyaltyGrade {
    ABSOLUTE,   # 臣服如山  91–100
    HIGH,       # 忠心耿耿  76–90
    MODERATE,   # 表面恭敬  61–75
    UNCERTAIN,  # 意圖未明  41–60
    LOW,        # 暗懷異志  21–40
    TRAITOR     # 潛在叛徒   0–20
}

enum LoyaltyEventType {
    COUNCIL_AGREE,       # 評議立場一致
    COUNCIL_DISAGREE,    # 評議立場相反
    REWARDED,            # 獲得封賞
    DEMOTED,             # 地位削弱
    IGNORED,             # 連續被忽視
    HIGH_NENGU,          # 年貢過高
    POST_WAR             # 戰後處置
}
```


## 7. 驗收準則（DoD 對應）

| 驗收項目 | 判定方式 |
|---------|---------|
| 忠誠數值邊界夾緊在 [0, 100] | 單元測試 |
| 未探查家臣顯示過期分級（非即時） | 手動測試 |
| 高階忍者才能揭露低忠誠 | 單元測試 |
| 連續 3 回合潛在叛徒觸發叛變事件 | 手動測試 |
| 500 名家臣全體結算 < 0.1 秒 | 效能計時 |
| UI 不顯示任何 0–100 數值 | Code Review |
