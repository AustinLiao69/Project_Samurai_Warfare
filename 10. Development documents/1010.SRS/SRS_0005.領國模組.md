## 1. 引言

### 1.1 目的
本文件定義領國模組（TerritorySystem）的軟體需求，涵蓋三層級領地架構、年貢系統、設施升級與資源管理，作為實作與驗收依據。

### 1.2 範圍
TerritorySystem 負責：
- 三層級領地（令制國 / 城 / 郡）的資料管理
- 郡類型（農業 / 商業 / 軍事）與資源產出
- 年貢比例設定與每回合收取
- 設施建設與升級（三類各三級）
- 叛亂風險計算
- 知行（封賞領地）邏輯

本 SRS **不涵蓋**戰後佔領領地的處置邏輯（見 SRS_0009），亦不涵蓋地圖視覺呈現（見 SRS_0002 地圖系統）。

### 1.3 術語定義

| 術語 | 說明 |
|------|------|
| 令制國 | 最高行政層級，包含多座城 |
| 城 | 第二層級，統治中心，含多個郡 |
| 郡 | 最基本行政單位，分三種類型 |
| 年貢 | 大名從郡徵收的資源（金錢/軍糧） |
| 知行 | 封賜給家臣管理的領地 |
| 統治衝擊 | 佔領後兵力/年貢/民心短暫下降的狀態 |

- `0005.領國模組.md` v1.1
- `0009.戰後處置模組.md` v3.0（統治衝擊）
- `PM008.MVP初始資料.md`（初始城等級與兵力）

## 2. 系統概述

### 2.1 領地層級架構

```
令制國 (Province)
└── 城 (Castle) × N
    └── 郡 (District) × M
        └── 設施 (Facility) × 最多 5
```

### 2.2 外部相依模組

| 相依模組 | 相依方式 | 用途 |
|---------|---------|------|
| IntrigueSystem | 呼叫 | 年貢過高時送出忠誠 delta |
| CoreSystem | 被呼叫 | 每回合年貢結算、設施升級 |
| MilitarySystem | 被呼叫 | 取得城等級（決定城兵上限） |
| PostWarSystem | 被呼叫 | 佔領後設定統治衝擊狀態 |
| RetainerSystem | 雙向 | 知行分配、家臣治理 |

## 3. 功能需求（SA 視角）

### 3.1 郡類型與基礎產出

**FR-TER-001**：郡分為三種類型，各有不同資源產出方向：

| 類型 | 主要產出 | 設施類型 |
|------|---------|---------|
| 農業郡 | 金錢 + 農業資源 | 農業設施 |
| 商業郡 | 金錢 + 商業資源 | 商業設施 |
| 軍事郡 | 軍糧 + 軍事資源 | 軍事設施 |

**FR-TER-002**：郡的基礎產出量由設施等級決定（詳見 3.3 設施升級）。

### 3.2 年貢系統

**FR-TER-003**：大名可對每個郡設定年貢比例（10%–50%）或固定年貢金額。

**FR-TER-004**：年貢每回合（月）自動結算，結算金額 = 郡產出 × 年貢比例。

**FR-TER-005**：忠誠度影響公式：

```
忠誠度下降 = (年貢比例 - 目標年貢比例) × 影響因子
```
- 目標年貢比例：視郡資源產量設定（建議預設值 30%）
- 影響因子：預設 0.5（待平衡調整）
- 忠誠度變化透過呼叫 `IntrigueSystem.apply_loyalty_delta()` 執行

**FR-TER-006**：若年貢超過郡產量上限，超出部分無法徵收（資源枯竭保護）。

**FR-TER-007**：叛亂風險公式：

```
叛亂風險 = (低忠誠度狀態下的年貢比例 / 高年貢設定比例) × 連續高年貢回合數
```
- 當叛亂風險 ≥ 100% 時，觸發叛亂事件（CoreSystem.trigger_event）

### 3.3 設施升級系統

**FR-TER-008**：每個郡最多 5 個設施，三大類各自升級路徑如下：

#### 農業設施
| 等級 | 名稱 | 升級費用 | 升級時間 | 累計費用 |
|------|------|---------|---------|---------|
| 1 | 村落 | — | — | — |
| 2 | 農村 | 100 金 | 1 月 | 100 金 |
| 3 | 農莊 | 300 金 | 2 月 | 400 金 |

#### 商業設施
| 等級 | 名稱 | 升級費用 | 升級時間 | 累計費用 |
|------|------|---------|---------|---------|
| 1 | 市集 | — | — | — |
| 2 | 商街 | 120 金 | 1 月 | 120 金 |
| 3 | 行會 | 360 金 | 2 月 | 480 金 |

#### 軍事設施
| 等級 | 名稱 | 升級費用 | 升級時間 | 累計費用 |
|------|------|---------|---------|---------|
| 1 | 駐屯地 | — | — | — |
| 2 | 兵營 | 150 金 | 1 月 | 150 金 |
| 3 | 軍陣營 | 400 金 | 2 月 | 550 金 |

**FR-TER-009**：升級為非即時，需消耗對應月數的建造時間（建造期間設施仍運作於舊等級）。

**FR-TER-010**：升級費用從大名金庫扣除，金庫不足時禁止升級。

**FR-TER-011**：高年貢期間設施升級速度下降（具體公式待平衡設計）。

### 3.4 知行（領地分封）

**FR-TER-012**：大名可將郡或城分封給家臣作為知行。分封後：
- 該領地的年貢按設定比例轉交家臣
- 家臣對該領地有治理權（可升級設施）
- 家臣的忠誠度受知行是否被撤回影響

**FR-TER-013**：撤回知行時，家臣忠誠度下降（透過 IntrigueSystem）。

### 3.5 城等級（Castle Level）

**FR-TER-014**：城池有等級（Lv1–Lv3+），決定城兵上限：

| 城等級 | 城兵上限 |
|-------|---------|
| Lv1 | 500 兵 |
| Lv2 | 1,000 兵 |
| Lv3 | 1,500 兵 |

> MVP 初始城資料依 PM008 設定：清洲城 Lv2、小牧山城 Lv1、岡崎城 Lv3、吉田城 Lv1。

## 5. 資料模型

```
Province (Resource)         # 令制國
├── id: String
├── name: String
├── faction_id: String
└── castle_ids: Array[String]

Castle (Resource)           # 城
├── id: String
├── name: String
├── province_id: String
├── faction_id: String
├── castle_level: int       # 1–5
├── current_garrison: int   # 城兵現況
├── assigned_retainer_id: String
└── district_ids: Array[String]

District (Resource)         # 郡
├── id: String
├── name: String
├── castle_id: String
├── district_type: DistrictType  # 農業/商業/軍事
├── base_output: int        # 基礎產出（金錢或軍糧）
├── nengu_rate: float       # 年貢比例 0.1–0.5
├── loyalty_value: int      # 郡民忠誠度（0–100，影響叛亂）
├── rebellion_risk: float   # 叛亂風險 0.0–1.0
├── assigned_retainer_id: String
└── facilities: Array[Facility]

Facility (Resource)         # 設施
├── id: String
├── facility_type: FacilityType   # 農業/商業/軍事
├── current_level: int      # 1–3
├── is_upgrading: bool
├── upgrade_turns_remaining: int
└── upgrade_cost_paid: int
```

## 6. GDScript 介面規格（PG 視角）

### 6.1 列舉型別

```gdscript
enum DistrictType {
    AGRICULTURAL,  # 農業郡
    COMMERCIAL,    # 商業郡
    MILITARY       # 軍事郡
}

enum FacilityType {
    FARM,          # 農業設施（村落→農村→農莊）
    MARKET,        # 商業設施（市集→商街→行會）
    BARRACKS       # 軍事設施（駐屯地→兵營→軍陣營）
}
```

### 6.2 系統節點

```gdscript
class_name TerritorySystem extends Node

# --- 查詢 ---
func get_district(id: String) -> District
func get_castle(id: String) -> Castle
func get_province(id: String) -> Province
func get_all_districts(faction_id: String) -> Array[District]
func get_castle_garrison_cap(castle_id: String) -> int

# --- 年貢 ---
func set_nengu_rate(district_id: String, rate: float) -> bool
    # 驗證 rate 在 [0.1, 0.5]，超出回傳 false
func collect_nengu(district_id: String) -> Dictionary
    # 回傳: {"gold": int, "supplies": int, "loyalty_delta": int}
func collect_all_nengu(faction_id: String) -> Dictionary
    # 全勢力年貢批次結算

# --- 叛亂風險 ---
func calculate_rebellion_risk(district_id: String) -> float
func check_rebellion_triggers(faction_id: String) -> Array[String]
    # 回傳叛亂風險 >= 1.0 的郡 ID 列表

# --- 設施 ---
func start_facility_upgrade(district_id: String, facility_type: FacilityType) -> Dictionary
    # 回傳: {"success": bool, "cost": int, "turns_required": int, "message": String}
func process_facility_upgrades(faction_id: String) -> Array[String]
    # 每回合推進建造進度，回傳本回合完工的設施 ID 列表
func get_facility_output(facility: Facility) -> int
func get_upgrade_cost(facility_type: FacilityType, current_level: int) -> int
func get_upgrade_time(facility_type: FacilityType, current_level: int) -> int

# --- 知行 ---
func grant_territory(district_or_castle_id: String, retainer_id: String) -> bool
func revoke_territory(district_or_castle_id: String) -> Dictionary
    # 回傳: {"success": bool, "loyalty_delta": int}
func get_total_output(district_id: String) -> Dictionary
    # 回傳: {"gold": int, "supplies": int}

# --- 城等級 ---
func get_castle_level_cap(castle_level: int) -> int  # 回傳兵力上限

# --- 治理 ---
func govern_district(district_id: String, action: String) -> Dictionary
    # action: "increase_output" | "improve_loyalty" | "upgrade_facility"
```

### 6.3 訊號

```gdscript
signal nengu_collected(faction_id: String, total_gold: int, total_supplies: int)
```

## 7. 驗收準則（DoD 對應）

| 驗收項目 | 判定方式 |
|---------|---------|
| 年貢比例限制在 10%–50% | 單元測試：超出值應被拒絕 |
| 設施升級費用和時間正確 | 單元測試 |
| 建造期間舊等級維持運作 | 手動測試 |
| 叛亂風險公式計算正確 | 單元測試 |
| 年貢超過上限時截斷 | 單元測試 |
| 城等級與兵力上限對應正確（MVP 四城） | 手動測試 |
| 全勢力年貢結算 < 0.5 秒 | 效能計時 |
