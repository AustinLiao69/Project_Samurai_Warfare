## 1. 引言

### 1.1 目的
本文件定義戰後處置模組（PostWarSystem）的軟體需求，涵蓋領地處置、家臣團收編、不滿度系統與忠誠恢復機制，作為實作與驗收依據。

### 1.2 範圍
PostWarSystem 負責：
- 佔領領地後的領地處置決策（沒收 / 投降保地）
- 家臣團收編選項（整團 / 分化 / 解散）
- 參戰家臣團的不滿度計算
- 收編武將的初始忠誠度設定
- 統治衝擊的逐月恢復機制

本 SRS **不涵蓋**戰鬥本身的流程（見 SRS_0006），亦不涵蓋忠誠度底層數值（見 SRS_0004）。

### 1.3 術語定義

| 術語 | 說明 |
|------|------|
| 統治衝擊 | 佔領後兵力/年貢/民心短暫下降到 50% 的狀態 |
| 不滿度 | 參戰家臣因功勞未獲酬報而產生的負面情緒值 |
| 整團收編 | 將敵方家臣團整體納入麾下 |
| 分化收編 | 精選部分武將，其餘成浪人 |
| 浪人 | 無主歸屬的武將（可被招募） |

## 2. 系統概述

### 2.1 流程總覽

```
[戰鬥勝利 / 佔領完成]
     ↓
[Step 1: 領地處置]
  ├─ 沒收領地 → [Step 2: 家臣團處理]
  └─ 投降保地 → [Step 3: 不滿度系統]
     ↓
[Step 4: 忠誠度恢復機制（每回合）]
```

### 2.2 外部相依模組

| 相依模組 | 相依方式 | 用途 |
|---------|---------|------|
| TerritorySystem | 呼叫 | 設定統治衝擊（兵力/年貢/民心 50%） |
| RetainerSystem | 呼叫 | 收編家臣、解散家臣團 |
| IntrigueSystem | 呼叫 | 設定收編武將初始忠誠度 |
| MilitarySystem | 被呼叫 | 戰鬥結束後觸發本系統 |

## 3. 功能需求（SA 視角）

### 3.1 Step 1 — 領地處置

**FR-PWS-001**：佔領敵方領地後，玩家必須在繼續進行前選擇領地處置方式：

**選項 A：沒收領地**
- 領地立即進入「大名直轄」狀態
- 觸發統治衝擊：
  - 城兵 → 降至原值 50%
  - 年貢收入 → 降至原值 50%
  - 民心 → 降至 50%
- 之後可決定是否封賞家臣或長期直轄
- 強制觸發 Step 2（家臣團處理）

**選項 B：投降保地**
- 原領主歸附，保留其領地與家臣團結構
- 若攻擊方家臣未獲賞賜 → 觸發 Step 3（不滿度系統）
- 不觸發 Step 2

### 3.2 Step 2 — 家臣團處理（僅在沒收領地後）

**FR-PWS-002**：原領主投降或被俘且領地被沒收時，玩家選擇家臣團處理方式：

**選項 1：整團收編**
- 原家臣團整體成為玩家直屬家臣團，保留原結構
- 收編武將初始忠誠度：依 FR-PWS-005 公式計算（基礎值低，存在叛亂風險）

**選項 2：分化收編**
- 玩家從名單中精選武將成為直臣，其餘成浪人
- 減少忠誠度問題，但戰力擴張較慢
- 玩家可指定最多 N 名武將（N 由可用直臣名額決定）

**選項 3：全部解散**
- 所有成員成為浪人（可被他勢力招募）
- 消除叛亂風險，但損失全部敵方戰力

**FR-PWS-003**：浪人武將進入全局浪人池，其他勢力（含 AI）可於後續招募。

### 3.3 Step 3 — 家臣不滿度系統

**FR-PWS-004**：不滿度觸發條件：攻擊方家臣團完成佔領，且玩家選擇「投降保地」未頒布任何賞賜。

**FR-PWS-004a**：不滿度計算公式：

```
不滿度增加 = 基礎值 × (家臣團參戰兵力 / 總參戰兵力)
```
- 基礎值：10（可調整）
- 每個參戰家臣團各自計算不滿度增加量

**FR-PWS-004b**：不滿度影響：
- 累積不滿度降低家臣忠誠度（每點不滿 = 忠誠 -1）
- 可能導致抗命、陽奉陰違、私自擴張等行為事件
- 賞賜（封地、金錢、寶物）可立即抵消不滿度
- 「承諾日後封賞」若失約：懲罰加倍（忠誠額外 ×2 下降）

**FR-PWS-004c**：頒布賞賜的 API 呼叫路徑：
1. `PostWarSystem.grant_reward()` → 
2. `IntrigueSystem.apply_loyalty_delta()` 套用忠誠恢復

### 3.4 Step 4 — 忠誠度初始化與統治恢復

**FR-PWS-005**：收編武將初始忠誠度公式：

```
初始忠誠度 = 50 - (敵對年數 × 5) + 賞賜加成
```
- 基礎值：50（可調整）
- 敵對年數：雙方交戰的年份數（每年 -5 點）
- 賞賜加成：封地（+15）、加官（+10）、金錢（+5）、寶物（+8）
- 夾緊在 [0, 100]

**FR-PWS-006**：統治衝擊恢復機制（僅適用沒收領地）：

```
每回合：民心 +5%
當民心 ≥ 80% → 兵力與年貢恢復正常
```
- 政治能力高的治理家臣：每點政治數值 > 60，恢復速度額外 +2%/回合
- 延遲恢復條件：暴政（年貢 > 40%）、戰亂、飢荒 → 恢復停止

## 4. 非功能需求

| 項目 | 需求 |
|------|------|
| 決策介面響應 | 玩家選擇後 < 0.5 秒完成狀態更新 |

├── promise_turns_remaining: int
└── promise_reward_type: String

## 6. GDScript 介面規格（PG 視角）

### 6.1 列舉型別

```gdscript
enum TerritoryDisposition {
    CONFISCATE,   # 沒收領地
    ALLOW_RETAIN  # 投降保地
}

enum RetainerDisposition {
    FULL_INTEGRATION,    # 整團收編
    SELECTIVE_INTEGRATION, # 分化收編
    DISSOLVE             # 全部解散
}

enum PostWarRewardType {
    LAND_GRANT,    # 封地
    PROMOTION,     # 加官
    GOLD,          # 金錢
    TREASURE,      # 寶物
    PROMISE        # 承諾日後封賞
}
```

### 6.2 系統節點

```gdscript
class_name PostWarSystem extends Node

# --- Step 1: 領地處置 ---
func initiate_post_war(territory_id: String, captured_faction_id: String) -> OccupationState
    # 戰鬥結束後呼叫，初始化佔領狀態

func decide_territory_disposition(state: OccupationState, disposition: TerritoryDisposition) -> Dictionary
    # 玩家選擇沒收或保地
    # 回傳: {"next_step": int, "immediate_effects": Dictionary}

func apply_occupation_shock(territory_id: String) -> void
    # 套用統治衝擊（兵力/年貢/民心 → 50%）

# --- Step 2: 家臣團處理 ---
func decide_retainer_disposition(
    state: OccupationState,
    disposition: RetainerDisposition,
    selected_retainer_ids: Array[String]  # 分化收編時使用
) -> Dictionary
    # 回傳: {"integrated": Array[String], "ronin": Array[String], "loyalty_values": Dictionary}

func set_initial_loyalty(retainer_id: String, enemy_years: int, rewards: Array[PostWarRewardType]) -> int
    # 依公式計算並設定初始忠誠度，回傳設定值

# --- Step 3: 不滿度 ---
func calculate_dissatisfaction(retainer_group_id: String, group_force: int, total_force: int) -> float
func apply_dissatisfaction(records: Array[DissatisfactionRecord]) -> void
    # 批次套用不滿度 → 呼叫 IntrigueSystem 扣忠誠

func grant_reward(retainer_group_id: String, reward_type: PostWarRewardType, value: int) -> Dictionary
    # 回傳: {"dissatisfaction_offset: float, "loyalty_delta": int}

func make_promise(retainer_group_id: String, reward_type: PostWarRewardType, turns: int) -> bool
func break_promise(retainer_group_id: String) -> Dictionary
    # 失約懲罰（加倍扣忠誠）

# --- Step 4: 統治恢復 ---
func process_recovery_turn(territory_id: String, governor_retainer_id: String) -> Dictionary
    # 每回合呼叫，回傳: {"morale_gain": float, "is_recovered": bool}
func check_recovery_blocked(territory_id: String) -> bool
    # 檢查是否因暴政/戰亂而停止恢復

# --- 浪人池 ---
func add_to_ronin_pool(retainer_ids: Array[String]) -> void
func get_ronin_pool() -> Array[Retainer]
func recruit_ronin(retainer_id: String, faction_id: String) -> bool
```

## 7. 驗收準則（DoD 對應）

| 驗收項目 | 判定方式 |
|---------|---------|
| 沒收後兵力/年貢/民心正確降至 50% | 單元測試 |
| 初始忠誠度公式計算正確 | 單元測試（驗算公式） |
| 不滿度公式計算正確 | 單元測試 |
| 失約懲罰為正常懲罰的 2 倍 | 單元測試 |
| 民心恢復至 80% 後兵力與年貢恢復正常 | 手動測試（推進 N 回合） |
| 政治高的家臣加速恢復 | 手動測試（比較不同政治值） |
