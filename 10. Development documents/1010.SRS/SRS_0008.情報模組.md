## 1. 引言

### 1.1 目的
本文件定義情報模組（IntelligenceSystem）的軟體需求，涵蓋忍者系統、六種任務類型、情報類型管理與防諜機制，作為實作與驗收依據。

### 1.2 範圍
IntelligenceSystem 負責：
- 忍者屬性管理（等級、技能、行動時長）
- 六種情報任務的派遣、成功率計算與結果處理
- 情報資料的儲存與有效期管理
- 防諜系統（敵方忍者偵測）
- 情報真假機制（誤導情報）
- 內政忍者部署（情報府）

本 SRS **不涵蓋**忠誠度的底層數值（見 SRS_0004），亦不涵蓋評議情報行動的 UI 場景流程（見 SRS_0007）。

### 1.3 術語定義

| 術語 | 說明 |
|------|------|
| 忍者 | 執行情報任務的特殊單位 |
| 任務 | 忍者被指派的一次情報行動 |
| 情報等級 | 刺探結果的深度（層層遞進） |
| 警戒度 | 目標勢力的防諜嚴密程度 |
| 情報府 | 養訓忍者的內部機構（可解鎖） |

## 2. 系統概述

### 2.1 模組定位
IntelligenceSystem 管理所有情報行動，並在任務完成後將結果推送給相關系統（忠誠度更新、地圖情報更新等）。

### 2.2 外部相依模組

| 相依模組 | 相依方式 | 用途 |
|---------|---------|------|
| IntrigueSystem | 呼叫 | 籠絡/離間成功時修改忠誠度 |
| TerritorySystem | 呼叫 | 破壞任務損壞目標設施 |
| RetainerSystem | 呼叫 | 暗殺成功後移除家臣 |
| MilitarySystem | 被呼叫 | 取得敵方兵力情報（刺探結果） |
| CouncilSystem | 被呼叫 | 評議情報行動觸發忍者派遣介面 |

## 3. 功能需求（SA 視角）

### 3.1 情報類型

**FR-INT-I-001**：可被收集與揭露的情報分六大類：

| 分類 | 子項目 | 特殊說明 |
|------|--------|---------|
| 外部情報 | 國力、軍力、內政等級（農/商/軍）、金糧存量 | 越強的勢力越難刺探 |
| 外交情報 | 與他國關係、盟約、潛在敵意 | 可從造謠/離間結果反推 |
| 軍事部署 | 各城兵力、將領配置、部隊移動 | 高階忍者才能刺探 |
| 家臣團情報 | 家臣忠誠、野心、政治立場 | 可暴露叛變危險 |
| 城內情報 | 城池設施、防衛強度、資源儲量 | 供破壞任務目標判斷 |
| 群眾情緒 | 民心士氣、是否受敵國影響 | 造謠/離間的依據 |

**FR-INT-I-002**：情報採層級制，每次刺探只揭露一層，需多次行動才能獲得深度情報（例：軍力評估 → 將領名單 → 部隊位置 → 移動路線）。

### 3.2 忍者系統

**FR-INT-I-003**：忍者屬性：

| 屬性 | 說明 |
|------|------|
| level | 等級（1–5），決定可執行任務範圍與基礎成功率 |
| specialization | 技能偏重（刺探/破壞/操縱/暗殺），對應任務有加成 |
| action_turns | 當前任務剩餘行動回合數 |
| is_available | 非任務中才可派遣 |
| identity_exposed | 身分是否已暴露（暴露後效力下降） |

**FR-INT-I-004**：忍者等級與任務限制：

| 等級 | 可執行任務 |
|------|-----------|
| 1–2 | 刺探（外部/外交）、造謠、破壞（低級目標） |
| 3 | 以上全部 + 刺探（家臣忠誠）、離間、籠絡 |
| 4 | 以上全部 + 刺探（軍事部署）、暗殺（低階武將） |
| 5 | 全部任務 + 暗殺大名 |

**FR-INT-I-005**：可派多名忍者協作同一任務（提升成功率），或分工（如一人造謠同時一人破壞）。

### 3.3 六種任務類型

**FR-INT-I-006**：任務規格完整定義：

| 任務 | 成功效果 | 失敗懲罰 | 特殊說明 |
|------|---------|---------|---------|
| 刺探 | 獲得目標特定情報（一層） | 無收穫 / 忍者被捕 / 外交惡化 | 情報有機率為假（見 3.6） |
| 暗殺 | 目標武將死亡 / 重傷 | 忍者被捕 / 外交惡化 | 高風險高報酬；Lv4 以上才可暗殺大名 |
| 籠絡 | 目標家臣忠誠提升 / 轉為內應 | 忍者身分暴露 / 家臣反感 | 需多次任務累積信任 |
| 離間 | 目標家臣忠誠下降 / 挑起內鬥 | 忍者被察覺 / 目標忠誠反升 | 民心低時成功率更高 |
| 造謠 | 目標民心下降 / 引起動亂 | 被識破 → 我方聲譽受損 | 可配合軍事行動使用 |
| 破壞 | 損壞目標設施 / 斷補給 / 降士氣 | 忍者被捕 / 敵方提升警戒 | 不需正面交戰即可削弱敵方 |

### 3.4 成功率計算

**FR-INT-I-007**：任務基礎成功率公式：

```
成功率 = 忍者基礎率（依等級） + 技能加成 + 協作加成 - 敵方警戒度 - 敵方防諜等級
```

| 忍者等級 | 基礎成功率 |
|---------|-----------|
| 1 | 40% |
| 2 | 55% |
| 3 | 65% |
| 4 | 75% |
| 5 | 85% |

| 技能偏重（對應任務） | 加成 |
|--------------------|------|
| 匹配任務類型 | +15% |
| 不匹配 | 0% |

**FR-INT-I-008**：警戒度影響：目標勢力警戒度每 +1 級，我方成功率 -10%（警戒度範圍：0–5）。

### 3.5 行動時長

**FR-INT-I-009**：任務所需回合數（月）：

| 任務 | 基礎回合數 | 距離影響 |
|------|-----------|---------|
| 刺探 | 1 回合 | +1/2 回合（遠距） |
| 造謠 | 1 回合 | +1 回合（遠距） |
| 離間 | 2 回合 | — |
| 破壞 | 1–2 回合 | 依設施等級 |
| 籠絡 | 3 回合 | — |
| 暗殺 | 2–3 回合 | 依目標等級 |

**FR-INT-I-010**：任務執行期間忍者不可再派遣（is_available = false）。

### 3.6 情報真假機制

**FR-INT-I-011**：刺探成功時，情報有 20% 機率為假（誤導情報），玩家無法直接得知真假，需靠多次驗證或高等級忍者。

**FR-INT-I-012**：玩家可選擇對同一目標再次刺探以驗證情報，第二次成功後若結果一致則可信度 + ；若矛盾則標注「情報存疑」。

### 3.7 防諜系統

**FR-INT-I-013**：我方可派武將設置防諜網（抵擋敵方刺探/籠絡），增加警戒度。

**FR-INT-I-014**：敵方忍者任務失敗時，有機率被我方防諜網捕獲，觸發情報（得知有忍者滲透）。

### 3.8 情報府（內政部署）

**FR-INT-I-015**：玩家可在本勢力建立「情報府」機構，養訓忍者提升整體等級上限。

**FR-INT-I-016**：玩家可向其他勢力雇用「浪人忍者」執行特定高風險任務（一次性，無法升級）。

## 4. 非功能需求

| 項目 | 需求 |
|------|------|
| 成功率計算速度 | 單次計算 < 0.01 秒 |
| 情報有效期 | 情報標注取得回合，過期後在 UI 加注「過期情報」 |
| 情報資料與主存檔同步 | 存檔時一併儲存已知情報 |

## 5. 資料模型

```
Ninja (Resource)
├── id: String
├── name: String
├── level: int               # 1–5
├── specialization: NinjaSpec
├── action_turns_remaining: int
├── is_available: bool
├── identity_exposed: bool
└── faction_id: String

NinjaMission (Resource)
├── id: String
├── ninja_ids: Array[String] # 執行任務的忍者（可多名）
├── mission_type: MissionType
├── target_id: String        # 目標（勢力/家臣/設施 ID）
├── turns_remaining: int
├── success_rate: float
└── status: MissionStatus    # PENDING / IN_PROGRESS / COMPLETED / FAILED

IntelReport (Resource)
├── id: String
├── report_type: ReportType
├── target_id: String
├── content: Dictionary      # 情報內容（依類型結構不同）
├── acquired_turn: int
├── is_false: bool           # 情報是否為假（玩家不可直接看到）
└── credibility: int         # 0–100，玩家可見的可信度指標
```

## 6. GDScript 介面規格（PG 視角）

### 6.1 列舉型別

```gdscript
enum NinjaSpec {
    ESPIONAGE,    # 刺探專精
    SABOTAGE,     # 破壞專精
    MANIPULATION, # 操縱（籠絡/離間）專精
    ASSASSINATION # 暗殺專精
}

enum MissionType {
    ESPIONAGE,    # 刺探
    ASSASSINATION,# 暗殺
    RECRUIT,      # 籠絡
    UNDERMINE,    # 離間
    RUMOR,        # 造謠
    SABOTAGE      # 破壞
}

enum MissionStatus {
    PENDING,
    IN_PROGRESS,
    COMPLETED,
    FAILED
}

enum ReportType {
    EXTERNAL,      # 外部情報
    DIPLOMATIC,    # 外交情報
    MILITARY,      # 軍事部署
    RETAINER,      # 家臣團情報
    CASTLE,        # 城內情報
    MORALE         # 群眾情緒
}
```

### 6.2 系統節點

```gdscript
class_name IntelligenceSystem extends Node

# --- 忍者管理 ---
func get_available_ninjas(faction_id: String) -> Array[Ninja]
func get_all_ninjas(faction_id: String) -> Array[Ninja]
func hire_ronin_ninja(level: int) -> Ninja   # 浪人忍者（一次性）
func can_execute_mission(ninja_id: String, mission_type: MissionType) -> bool

# --- 任務派遣 ---
func assign_mission(ninja_ids: Array[String], mission_type: MissionType, target_id: String) -> Dictionary
    # 回傳: {"success": bool, "mission_id": String, "turns_required": int, "estimated_success_rate": float}
func cancel_mission(mission_id: String) -> bool

# --- 成功率計算 ---
func calculate_success_rate(ninja_ids: Array[String], mission_type: MissionType, target_id: String) -> float
func get_target_alert_level(target_faction_id: String) -> int   # 0–5

# --- 任務處理（每回合呼叫）---
func process_missions(faction_id: String) -> Array[Dictionary]
    # 推進所有進行中任務，回傳本回合完成的結果列表
    # 每個結果: {"mission_id": String, "success": bool, "result_data": Dictionary}
func resolve_mission_result(mission: NinjaMission) -> Dictionary
    # 內部：依任務類型處理結果（修改忠誠度、刪除家臣等）

# --- 情報管理 ---
func store_report(report: IntelReport) -> void
func get_reports(faction_id: String, report_type: ReportType) -> Array[IntelReport]
func get_report_for_target(target_id: String, report_type: ReportType) -> IntelReport
func is_report_expired(report: IntelReport, current_turn: int, expiry_turns: int) -> bool

# --- 防諜 ---
func set_counterintel(territory_id: String, retainer_id: String) -> bool
func raise_alert_level(faction_id: String, amount: int) -> void
func detect_enemy_ninja(enemy_mission: NinjaMission) -> bool

# --- 情報府 ---
func establish_intel_office(territory_id: String) -> bool
func train_ninja(ninja_id: String) -> bool   # 升一級，需費用與時間
```

## 7. 驗收準則（DoD 對應）

| 驗收項目 | 判定方式 |
|---------|---------|
| 六種任務類型均可派遣 | 手動測試 |
| 忍者等級限制正確（Lv1 不可暗殺大名） | 單元測試 |
| 成功率公式計算正確 | 單元測試 |
| 任務執行期間忍者不可再派遣 | 手動測試 |
| 刺探成功情報有 20% 假情報機率（隨機） | 統計測試（100 次取樣） |
| 任務結果正確觸發相關系統（忠誠度/設施損壞等） | 手動測試 + Log |
