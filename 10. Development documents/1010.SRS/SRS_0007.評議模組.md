## 1. 引言

### 1.1 目的
本文件定義評議模組（CouncilSystem）的軟體需求，涵蓋評議觸發方式、流程、家臣表態生成與忠誠度影響，作為實作與驗收依據。

### 1.2 範圍
CouncilSystem 負責：
- 評議的兩種觸發方式（玩家主動 / 系統自動）
- 四類評議類別的議題管理
- 家臣表態的生成邏輯（依個人利益、派系、性格）
- 大名裁決機制
- 評議結果對忠誠度與派系勢力的影響

本 SRS **不涵蓋**忠誠度數值的底層計算（見 SRS_0004），亦不涵蓋評議中情報行動的忍者場景（由 IntelligenceSystem 負責）。

### 1.3 術語定義

| 術語 | 說明 |
|------|------|
| 評議 | 大名召集重臣共同議決重大政策的機制 |
| 表態 | 家臣對議題的立場（贊成 / 反對 / 中立） |
| 裁決 | 大名的最終決定（執行 / 否決） |
| 派系立場 | 家臣所屬派系對此類議題的預設傾向 |

- `0004.封建權謀模組.md` v1.0（派系）
- `0010.派系思想模組.md` v0.3（政務分派傾向）

## 2. 系統概述

### 2.1 模組定位
CouncilSystem 是大名決策的正式化機制，作為政治博弈的核心舞台。評議結果直接影響整個派系的忠誠度分布。

### 2.2 外部相依模組

| 相依模組 | 相依方式 | 用途 |
|---------|---------|------|
| IntrigueSystem | 呼叫 | 評議後忠誠度 delta 寫入 |
| FactionSystem | 讀取 | 取得家臣派系立場，影響表態生成 |
| RetainerSystem | 讀取 | 取得參與者列表（家老、分封領主等） |
| MilitarySystem | 被呼叫 | 軍事評議自動觸發 |
| TerritorySystem | 被呼叫 | 行政評議自動觸發（大稅改、新建城） |
| CoreSystem | 被呼叫 | 特別評議自動觸發（繼承人任命等） |

## 3. 功能需求（SA 視角）

### 3.1 評議觸發方式

**FR-COU-001**：評議有兩種觸發模式並存：

| 觸發模式 | 入口 | 說明 |
|---------|------|------|
| 玩家主動 | 政務選單第 6 項「評定會議」 | 玩家自選議題 |
| 系統自動 | 玩家執行特定行動時跳出 | 強制評議後才可執行 |

**FR-COU-002**：系統自動觸發條件：

| 行動類型 | 觸發條件 |
|---------|---------|
| 行政評議 | 年貢率變動 ≥ 10%，或新建城池 |
| 外交評議 | 宣戰、結盟、和議、聯姻 |
| 軍事評議 | 出兵兵力 > 本家總兵力 30%，或任命出征主將 |
| 特別評議 | 繼承人任命、城級以上領地分封、頒布全國法令 |

> 小規模行為（單郡年貢調整、郡級設施升級）**不觸發**評議，玩家可直接執行。

### 3.2 評議類別

**FR-COU-003**：四類評議議題：

| 類別 | 典型議題 |
|------|---------|
| 行政評議 | 開墾農田、修築城牆、獎勵商業、徵收稅賦 |
| 外交評議 | 聯姻、結盟、宣戰、和議、上洛 |
| 軍事評議 | 出兵攻打、調兵遣將、防禦布署 |
| 特別評議 | 繼承人任命、領地分封、重大法令 |

### 3.3 召集對象

**FR-COU-004**：評議自動生成參與者名單，來源：
- 直屬家老（所有宿老/家老）
- 分封領主（依地位與資歷）
- 特殊職位（軍奉行、政務奉行、奉行衆等，MVP 簡化為「高職位家臣」）

### 3.4 家臣表態生成

**FR-COU-005**：每位與會家臣根據以下因素生成表態（贊成 / 反對 / 中立）：

| 因素 | 說明 |
|------|------|
| 個人利益 | 封地、稅收影響、軍權 |
| 派系立場 | 本家、外樣、反對派的傾向 |
| 政務分派 | 武功派傾向贊成軍事；文治派傾向反對戰爭 |
| 性格特質 | 保守、進取、謹慎、冒險（影響立場強度） |
| 個人忠誠度 | 忠誠越低，越傾向反對大名立場 |

**FR-COU-006**：表態附隨生成文字意見（從預設文字池中選取並填入角色名），格式範例：
> 「宿老・山內盛員：『當前糧草不足，恐難長期作戰，望殿下三思。』（反對）」

**FR-COU-007**：文字池需依評議類別、立場（贊成/反對）分類儲存（可外部 JSON 資料表管理）。

### 3.5 大名裁決

**FR-COU-008**：玩家選擇「執行」或「否決」議案。裁決影響：

| 狀況 | 忠誠度影響 |
|------|---------|
| 多數贊成 → 執行 | 贊成者忠誠度不受負面影響 |
| 多數反對 → 強行執行 | 反對者忠誠度下降（-5 ~ -20） |
| 多數贊成 → 被否決 | 贊成者忠誠度下降（-5 ~ -20） |
| 任何裁決後家臣獲封賞 | 忠誠度上升（+10 ~ +25） |

**FR-COU-009**：忠誠度修改統一透過 `IntrigueSystem.apply_batch_deltas()` 執行。

### 3.6 派系勢力變動

**FR-COU-010**：評議結束後，派系勢力依多數/少數表態調整：
- 贊成派系若裁決與立場一致 → 勢力值 +
- 反對派系若被強行否決 → 勢力值 -

**FR-COU-011**：大名連續偏袒同一派系（連續 3 次評議支持同一派），其他派系集體忠誠度額外 -10 ~ -20。

### 3.7 情報行動評議

**FR-COU-012**：評議類別「情報行動」觸發特殊場景：切入密談室畫面並顯示忍者首領，進入 IntelligenceSystem 的任務指派介面。

## 4. 非功能需求

| 項目 | 需求 |
|------|------|
| 表態生成速度 | 20 位參與者的表態與文字生成 < 0.5 秒 |
| 文字池 | 每類別至少 10 條文字模板（MVP 可先用 5 條） |
| 可擴充性 | 文字池從外部資料讀取，不硬編碼在腳本中 |

## 5. 資料模型

```
CouncilSession (Resource)       # 一次評議會議
├── id: String
├── council_type: CouncilType
├── topic: String               # 議題描述
├── participants: Array[String] # 參與者 Retainer ID
├── stances: Dictionary         # {retainer_id: CouncilStance}
├── opinion_texts: Dictionary   # {retainer_id: String} 文字意見
├── majority_stance: CouncilStance
├── player_decision: bool       # true=執行, false=否決
└── loyalty_changes: Array[Dictionary]  # 裁決後的忠誠 delta 列表

CouncilStance: enum { AGREE, OPPOSE, NEUTRAL }
CouncilType: enum { ADMINISTRATION, DIPLOMACY, MILITARY, SPECIAL }
```

## 6. GDScript 介面規格（PG 視角）

### 6.1 列舉型別

```gdscript
enum CouncilType {
    ADMINISTRATION,  # 行政評議
    DIPLOMACY,       # 外交評議
    MILITARY,        # 軍事評議
    SPECIAL          # 特別評議
}

enum CouncilStance {
    AGREE,    # 贊成
    OPPOSE,   # 反對
    NEUTRAL   # 中立
}

enum PersonalityType {
    CONSERVATIVE,  # 保守
    AGGRESSIVE,    # 進取
    CAUTIOUS,      # 謹慎
    ADVENTUROUS    # 冒險
}
```

### 6.2 系統節點

```gdscript
class_name CouncilSystem extends Node

# --- 觸發評議 ---
func open_council(council_type: CouncilType, topic: String, topic_params: Dictionary) -> CouncilSession
    # 主動召開評議，回傳會議物件
func check_auto_trigger(action_type: String, action_params: Dictionary) -> bool
    # 判斷行動是否自動觸發評議，供 CoreSystem 在執行行動前呼叫
func auto_trigger_council(action_type: String, action_params: Dictionary) -> CouncilSession
    # 自動觸發並回傳會議物件

# --- 參與者 ---
func generate_participants(council_type: CouncilType, faction_id: String) -> Array[String]
    # 依評議類型決定參與者（家老、分封領主等）

# --- 表態生成 ---
func generate_stance(retainer_id: String, session: CouncilSession) -> CouncilStance
    # 依個人利益、派系、性格、忠誠度計算表態
func generate_opinion_text(retainer_id: String, stance: CouncilStance, council_type: CouncilType) -> String
    # 從文字池選取模板並填入角色名
func get_majority_stance(session: CouncilSession) -> CouncilStance

# --- 裁決 ---
func player_decide(session: CouncilSession, execute: bool) -> Dictionary
    # 玩家決定執行或否決，回傳影響摘要
    # 回傳: {"loyalty_changes": Array[Dictionary], "faction_changes": Dictionary, "message": String}
func apply_council_results(session: CouncilSession) -> void
    # 呼叫 IntrigueSystem 套用忠誠度變化，更新派系勢力

# --- 連續偏袒檢查 ---
func check_consecutive_bias(faction_id: String, favored_group_id: String) -> bool
    # 若連續 3 次偏袒同一派系，回傳 true，觸發集體忠誠下降

# --- 情報行動 ---
func open_intelligence_action() -> void
    # 觸發密談室場景，交由 IntelligenceSystem 接管
```

### 6.3 訊號

```gdscript
signal council_opened(session: CouncilSession)
signal stance_generated(retainer_id: String, stance: CouncilStance)
signal player_decided(session: CouncilSession, execute: bool)
signal council_closed(session: CouncilSession)
signal bias_penalty_triggered(disfavored_group_id: String)
```

## 7. 驗收準則（DoD 對應）

| 驗收項目 | 判定方式 |
|---------|---------|
| 年貢變動 ≥ 10% 自動觸發評議 | 手動測試 |
| 出兵 > 30% 自動觸發軍事評議 | 手動測試 |
| 強行執行反對議案時反對者忠誠下降 | 手動測試 + Log |
| 文字意見從資料池動態生成 | Code Review |
| 20 位參與者表態生成 < 0.5 秒 | 效能計時 |
| 連續 3 次偏袒同一派系觸發懲罰 | 手動測試 |
